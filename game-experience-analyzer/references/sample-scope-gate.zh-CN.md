# Sample Scope Gate / 样本边界门

这个文件定义“当前材料能判断什么、不能判断什么”。每次输出报告时，正文第一段之后或第一张表必须先写样本边界门，不能先给总评。

## 必须先输出的四项

| 字段 | 必填规则 | 示例 |
| --- | --- | --- |
| `sample_boundary` | 写清输入材料、数量、时长、可访问状态、是否完整流程。 | `3 张截图 + 1 条 30 秒 PV，可访问但没有实机操作段` |
| `supported_judgment_scope` | 只列当前样本能支撑的判断。 | `可判断首屏信息层级、PV 首秒钩子、可传播峰值` |
| `unsupported_judgment_scope` | 明确写不能判断的内容。 | `不能判断操作手感、D7 留存、完整商业化深度` |
| `key_unknowns` | 列会改变结论的未知项，不要猜满。 | `真实战斗输入反馈、首小时功能解锁节奏、Steam 页面转化数据` |

硬规则：

- 如果用户问题超出样本能力，仍然要回答，但必须把该部分标成 `unsupported_by_sample` 或 `uncertain`。
- 置信度上限由样本决定。截图不能给“操作手感高置信判断”，PV 不能给“销量/流水确定预测”。
- 样本边界不等于免责。边界后仍要给出可执行诊断、补料清单和验证计划。

## 输入材料能力矩阵

| 输入材料 | 能支持的判断 | 不能支持的判断 | 必要证据 | 置信度上限 |
| --- | --- | --- | --- | --- |
| 截图 `screenshot` | UI 层级、视觉焦点、可见功能入口、静态奖励表达、文案承诺、静态 Hook/Link | 操作手感、等待时间、战斗节奏、循环闭合、动态反馈强度、真实留存 | `image_id`、`region`、`visible_text`、画面元素 | 静态 UI 可到 `high`；动态判断最高 `low` |
| PV/宣传片 `trailer_pv` | 首 3-6 秒钩子、卖点复述、可玩性证明是否出现、差异化表达、可传播峰值、平台/渠道适配假设 | 销量、流水、下载量、D1/D7、完整核心循环、长期内容供给质量 | `timestamp`、`frame_id`、画面/字幕、片段结构 | 热度潜力最高 `medium-high`；商业结果最高 `low` |
| 买量素材 `paid_creative` | 创意钩子、CTR 假设、素材误导风险、首屏可读性、落地页承接问题 | 自然量口碑、长期留存、完整付费深度、真实 LTV | 素材秒点、文案、CTA、落地页可见信息 | CTR 假设最高 `medium`；留存/LTV 最高 `low` |
| 本地录屏 `local_recording` | 时间轴、教程打断、功能暴露/解锁/首用、奖励节奏、循环是否初步闭合、可见失败/困惑信号 | 设备外体感、未录到的后续系统、长期留存、真实付费转化 | `timestamp`、事件类型、关键帧、玩家操作上下文 | 可见流程可到 `high`；未覆盖系统最高 `low` |
| 首小时录像 `first_hour_recording` | 首小时留存钩子、认知负担、首轮循环、D1 目标承接假设、商业化首次打断风险 | D7/D30、长期经济、版本后期内容、最终付费深度 | 阶段时间轴、功能账本、奖励/失败/商业化事件 | 首小时判断可到 `high`；长期判断最高 `medium-low` |
| 完整通关复盘 `full_run_review` | critical path、全局 pacing、关卡复用、Boss/终局反馈、finish intent、叙事-机制闭合 | 市场销量、玩家总体口碑、未体验分支、LiveOps 长线 | 通关范围、章节/关卡时间轴、关键问题证据 | 流程体验可到 `high`；市场判断最高 `low` |
| 视频链接不可访问 `inaccessible_video_url` | 页面标题、封面、简介、评论区可见信息、访问阻塞事实 | 视频时间轴、具体玩法、动态节奏、画面细节、剧情/机制事实 | `url`、`accessed_at`、错误状态、可见页面证据 | 页面层判断最高 `medium`；视频内容判断必须 `unsupported` |
| 商店页/Steam 页面 `store_page` | 首屏转化、标签/卖点、截图顺序、愿望单 CTA、系统需求/价格/评价可见信号 | 实机手感、真实留存、购买转化率、未公开销量 | 页面区块、可见文案、截图/视频帧、评价摘要 | 页面转化诊断可到 `medium-high`；销量最高 `low` |

## 边界升级规则

| 用户目标 | 当前样本不足时的处理 |
| --- | --- |
| 判断手感 | 要求本地录屏、手柄/键鼠输入展示、失败/成功反馈片段；截图只能判断反馈可读性。 |
| 判断销量/流水 | 只能给热度潜力和验证指标；需要商店页数据、投放数据、预约/愿望单/评论/转化数据。 |
| 判断首小时留存 | 需要至少连续 30-60 分钟新号流程或可定位阶段片段；PV 只能给留存钩子假设。 |
| 判断完整单机节奏 | 需要完整通关复盘或章节级材料；10 分钟片段只能判断 `first_segment` 或 `level_slice`。 |
| 判断商业化打断 | 需要商业化入口出现的时间点、前后玩家目标和奖励/付费文案；只看到商店图标不能判断打断强度。 |

## 最小示例

```yaml
sample_scope_gate:
  sample_boundary: "1 条 42 秒 PV，可访问；含 6 秒实机 UI，但没有连续操作流程。"
  supported_judgment_scope:
    - "首 3-6 秒钩子是否清楚"
    - "卖点是否能被一句话复述"
    - "PV 是否出现可玩性证明"
  unsupported_judgment_scope:
    - "不能判断操作手感"
    - "不能预测确定销量、流水或 D7 留存"
    - "不能证明完整核心循环闭合"
  key_unknowns:
    - "真实首小时流程"
    - "Steam 页面愿望单转化"
    - "Demo 完播/试玩转化"
```
