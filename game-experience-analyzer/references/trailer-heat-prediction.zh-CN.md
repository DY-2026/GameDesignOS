# PV / 宣传片热度潜力预测方法

这个文件用于用户要求分析游戏 PV、宣传片、预告片、实机宣传片、买量素材，或问“这个游戏能不能火 / 会不会火 / 有没有爆款潜力”。

## 1. 预测边界

不要输出确定性销量、流水或下载量预测。正确输出是：

- `heat_potential_tier`：爆款候选 / 强圈层爆款 / 中等待验证 / 高风险 / 证据不足。
- `confidence`：高 / 中 / 低。
- `why_now`：为什么现在可能有热度。
- `why_not`：为什么可能只热不转化。
- `validation_plan`：下一步用什么数据验证。

PV 只能证明“传播素材和产品承诺是否有潜力”，不能证明最终成品质量、留存、口碑、商业化或长期销量。

## 2. VOI 门

联网调研只在会改变结论时做：

| 信息 | 何时值得查 | 会改变什么 |
| --- | --- | --- |
| 官方页面、商店页、发售平台 | PV 里没有发售、平台、品类信息 | 平台适配、转化路径、愿望单/预约建议 |
| 当前播放、弹幕、评论、转发、收藏、热搜 | 用户问“能不能火”或“现在热度怎么样” | 热度潜力分层与置信度 |
| 竞品近期表现 | PV 的品类或题材依赖市场窗口 | 受众上限、差异化、风险 |
| 前作数据/口碑 | 续作、系列作、IP 作 | 冷启动难度、信任红利、反噬风险 |
| 平台算法/素材规范 | 用户要投放或上商店页 | 首秒、封面、CTA、素材长度和验证指标 |

不值得查：泛泛的行业新闻、无法追溯的数据、和本 PV 卖点无关的竞品。

## 3. 七层判断

### 3.1 首 3-6 秒钩子

检查：

- 第一眼是否看懂类型、题材或强视觉。
- 是否有异常物、冲突、敌人、角色、动作或高概念。
- 是否能无声理解。
- 标题/封面/第一镜是否匹配，避免点击后期待落空。

### 3.2 一句话卖点

看观众能否复述：

```text
这是一款给谁玩的、以什么题材/玩法为核心、哪里和同类不同的游戏。
```

如果只能复述“画面很强”“很有感觉”，说明传播可以热，但转化可能弱。

### 3.3 可玩性可读性

PV 要区分：

- 实机玩法：操作、敌人、规则、UI、失败/成功反馈可见。
- 实机演出：引擎内画面强，但玩法规则不清。
- CG 氛围：题材和情绪强，但产品承诺弱。
- 买量素材：强冲突强节奏，但可能和实际游戏割裂。

“实机”标签不能代替玩法证据。必须写清 PV 证明了什么，没证明什么。

### 3.4 差异化与品类受众

判断：

- 题材是否足够稀缺。
- 视觉符号是否一眼可认。
- 核心玩法是否落在已有大受众品类。
- 是否存在“高讨论但低购买”的错配。
- 是否有跨圈层桥梁：文化符号、角色、动作爽点、梗、争议、技术展示、前作信任。

### 3.5 传播峰值

记录可传播点：

- 可截图：角色、怪物、场景、UI、金句、反差画面。
- 可二创：梗、台词、角色表情、动作姿态、音乐节奏。
- 可讨论：题材解读、技术争议、玩法争议、世界观猜测。
- 可转化：看完后知道去哪里预约、加愿望单、关注账号或试玩。

### 3.6 平台与渠道适配

| 平台/渠道 | 重点 |
| --- | --- |
| Steam / 主机 | 品类标签、实机可信度、商店页、愿望单、Demo、媒体节点 |
| App Store / Google Play | 前几秒可视吸引、截图/视频首屏、功能和价值表达、A/B 测试 |
| B站 | 弹幕点、二创点、前作/UP主反应、评论情绪、硬核细节拆解 |
| 抖音 / TikTok | 首 3-6 秒、竖屏重剪、强字幕、明确冲突、短 CTA |
| TapTap / 国内社区 | 预约、评分预期、开发者可信度、测试资格、社区互动 |

### 3.7 转化断点

常见“热但不火”：

- PV 只有氛围，没有玩法承诺。
- 画面强，但无法说清和同类区别。
- 前作信任很强，但本作展示的信息量过少，引发空转争议。
- 传播强，但没有商店页、预约、愿望单或测试承接。
- 题材破圈，但核心品类门槛高，实际购买人群收窄。

### 3.8 前瞻机会叠加

当用户问“能不能火”或“值不值得做”时，PV 判断需要叠加前瞻机会镜头：

- 机会类型：这是结构创新、内容创新、题材创新、投放创新、组合创新，还是反转窗口。
- 市场窗口：处于源头萌芽、跟进窗口、验证窗口、红海，还是反转窗口。
- 用户情绪：玩家现在在奖励什么，厌倦什么。
- 转化路径：热度是否能导向预约、愿望单、Demo、测试资格或商店页。
- 最小验证：下一步是重剪素材、做可玩 Demo、做商店页 A/B，还是做小样本试玩。
- Kill 条件：如果只有讨论没有行动，或只有题材好奇没有玩法复述，就要降低投入。

## 4. 评分卡

| 维度 | 1 分 | 3 分 | 5 分 |
| --- | --- | --- | --- |
| 首秒钩子 | 慢热或看不懂 | 有视觉吸引但卖点后置 | 3-6 秒内完成题材/冲突/奇观建立 |
| 卖点清晰 | 只能说“好看” | 能说题材但玩法不清 | 能复述品类、核心卖点和差异 |
| 可玩性可信 | 基本无玩法证据 | 有片段但规则不清 | 动作、反馈、系统或目标可见 |
| 差异化 | 像同类换皮 | 有局部符号或题材差异 | 有强可识别符号和品类新鲜感 |
| 情绪峰值 | 无峰值 | 有单点记忆 | 多个可截图、可复述、可讨论瞬间 |
| 平台适配 | 和投放/商店不匹配 | 需重剪或补信息 | 当前素材天然适配目标渠道 |
| 转化承接 | 看完不知道去哪 | 有关注但弱 CTA | 明确预约/愿望单/试玩/关注路径 |

## 5. 热度潜力分层

| 分层 | 标准 |
| --- | --- |
| 爆款候选 | 强首秒、强差异化、玩法或题材可读、平台承接清楚，并已有或可快速验证外部热度信号 |
| 强圈层爆款 | 在特定品类/文化/平台圈层极强，但大众转化仍需补玩法或降低门槛 |
| 中等待验证 | 有卖点，但 PV 证据不足；需要测试封面、首 30 秒、商店页或 Demo |
| 高风险 | 传播和转化断裂，或素材承诺与实际产品可能错配 |
| 证据不足 | 链接不可访问、只能看到标题/封面，或样本不是宣传片主体 |

## 6. 必须输出的字段

| 字段 | 要求 |
| --- | --- |
| `trailer_type` | CG / in-engine / gameplay_trailer / vertical_ad / mixed / unknown |
| `first_3_6_seconds_hook` | 首秒证据和判断 |
| `one_sentence_pitch` | 观众可复述的一句话卖点 |
| `proof_of_play` | PV 对玩法、系统、反馈证明到什么程度 |
| `shareability` | 可截图、可二创、可争议、可讨论点 |
| `platform_channel_fit` | Steam/主机/移动/B站/抖音/TikTok/TapTap 等适配 |
| `heat_potential_tier` | 爆款候选 / 强圈层爆款 / 中等待验证 / 高风险 / 证据不足 |
| `confidence` | 高 / 中 / 低 |
| `key_uncertainties` | 影响判断的未知项 |
| `validation_plan` | CTR、前 30 秒留存、完播率、评论情绪、愿望单/预约转化、Demo 转化等 |
| `foresight_overlay` | 机会类型、窗口阶段、剩余窗口估计、Go/No-Go、最小验证截止和 Kill 条件 |

## 7. 外部来源校准

这些来源用于校准，不替代样本观察：

- TikTok Ads 官方建议把 hook 放在前 6 秒，并在前 3 秒介绍内容主张。来源：<https://ads.tiktok.com/help/article/creative-best-practices>
- YouTube audience retention 用 intro、top moments、spikes、dips 判断视频哪里留住或丢失观众。来源：<https://support.google.com/youtube/answer/9314415>
- Apple App Store 建议 app preview 前几秒要有视觉吸引力，并可测试图标、截图和预览视频。来源：<https://developer.apple.com/app-store/product-page/>
- Steam visibility 文档提醒标签、愿望单、销量、折扣通知、New Releases 等对 Steam 发行路径有不同作用。来源：<https://partner.steamgames.com/doc/marketing/visibility>
