# 游戏体验样本分析工作流

本工作流支持三类来源：

- 截图：适合分析首屏承诺、UI 层级、入口拥挤、奖励表达、长期目标露出和静态吸引力。
- 本地录屏：适合分析完整时间轴、节奏、教程打断、功能暴露/解锁/首用、循环建立、玩法机制和峰值。
- 视频链接：适合分析公开视频、云端录屏、平台视频和竞品片段；必须先确认链接可访问范围。

不要把三类来源混成同一种证据。截图没有时间轴，不能强行判断节奏；视频链接如果不可访问，不能编造内容；本地录屏如果缺字幕或画面模糊，要把低置信度判断标为 `uncertain`。

## 标准链路

1. 项目设定：明确业务目标、样本范围、时间预算、输出粒度。
2. 素材整理：收录截图、本地录屏、视频链接、版本说明、对标对象和命名规则。
3. 证据观察：截图建立画面证据表；录屏/视频按分钟或阶段标注事件、系统开启、节奏变化、峰值节点。
4. 框架建模：先选分析模式，再映射到对应镜头；四步法只在前期体验或用户点名时作为主镜头。
5. 原则抽象：把案例判断提升为可复用的设计原则。
6. 设计翻译：把原则落成可执行动作、优先级和验证方法。
7. 文档包装：输出 Markdown、DOCX、PPT 或结构化 JSON。
8. 最终 QA：检查证据、口径、版式、文件命名、可读性和渲染状态。

## 五层交付结构

| 层级 | 内容 | 价值 |
| --- | --- | --- |
| 输入层 | brief、截图、本地录屏、视频链接、版本说明、对标对象 | 保证判断回到原始上下文 |
| Agent 层 | 总控、观察、建模、方案、包装 QA | 明确分工和 done 定义 |
| Skill 层 | 多模态观察、时序标注、系统拆解、框架映射、文档包装 | 把能力沉淀成复用组件 |
| Tool 层 | 图片识别、视频抽帧、链接访问、OCR、转写、表格、文档生成、渲染 QA | 保留可回溯痕迹 |
| 输出层 | 报告、结构化 JSON、证据清单、模板快照 | 让结论可复审、可横向比较 |

## Agent 角色

| Agent | 核心职责 | 输出 |
| --- | --- | --- |
| 总控 Agent | 明确目标、口径、节奏和交付标准 | 任务清单、阶段定义、收口意见 |
| 观察 Agent | 按来源类型识别画面、事件、界面、节奏和情绪拐点 | 画面证据表、时间轴观察稿、关键帧、事件列表 |
| 建模 Agent | 把观察映射到分析模式与系统结构 | 模式路由、系统分类、问题归位 |
| 方案 Agent | 把诊断翻译成设计动作 | 优化建议、优先级、应用模板 |
| 工作流 Agent | 整理 SOP、模板、输入输出与质量门 | 流程图、职责表、模板清单 |
| 包装 QA Agent | 成稿、版式、证据核对和最终复审 | 正式稿、批注版、通过记录 |

## 最小项目目录

```text
01_input/
02_observation/
03_modeling/
04_proposal/
05_output/
```

`01_input` 放 brief、版本说明、截图、本地录屏、视频链接和对标清单。`02_observation` 放画面证据表、关键帧、时间轴稿、截图标注和联系图。`03_modeling` 放模式路由、分析镜头、系统树和问题归位表。`04_proposal` 放原则清单、机会池和优先级。`05_output` 放最终报告、PPT、摘要版和批注版。

## 质量门

- 证据门：每个结论都能追溯到原始素材。
- 抽象门：案例性判断已经被提升为可复用原则。
- 翻译门：原则被落实为可执行动作、优先级和验证方法。
- 包装门：最终交付件经过版式与渲染检查，能直接发送或评审。

## 最小执行版

如果时间有限，只保留五件事：

1. 一份清晰 brief。
2. 一份时间轴观察稿。
3. 一张分析模式与镜头路由表。
4. 按当前模式输出对应评分或判断表。
5. 一份设计原则和改法清单。
6. 一个经过 QA 的最终交付件。
