# 系统设计审查镜头

这个引用把系统设计判断压缩成适合截图、录屏、PV/宣传片、视频链接分析的审查镜头。它不替代其它分析模式；当前模式决定本镜头是否需要启用。

## 先判品类

先识别样本属于哪类主循环，再套对应问题。不要把所有游戏都塞进同一套“新手期”话术。

常见路由：

- 放置/轻量成长：点击、开箱、抽取、装备替换、离线收益、福利广告、排行榜和开服目标。
- 幸存者/RPG：战斗生存压力、资源搜集、角色成长、地图推进、敌群节奏和任务目标。
- RPG / ARPG：战斗动词、敌人语法、装备成长、关卡节奏。
- Roguelike：单局构筑、随机性、失败复盘、局内外成长边界。
- 模拟经营：生产链、订单、装饰、NPC/社交、资源瓶颈。
- X+SLG：表层是经营、放置、卡牌、RPG 或生存，长线由大地图、联盟、行军、领地、战报、赛季竞争承接；例如“模拟经营+SLG”。
- 卡牌/Gacha：抽取、阵容、养成材料、保底、角色目标。
- SLG/4X：地图、领地、联盟、时间资源、战报和长期竞争。
- MOBA/竞技：局内经济、对抗目标、技能可读性、团队协作、胜负反馈。

如果证据不足，写明“按当前可见样本，暂按 X 类处理”，并列出替代可能。

遇到复合品类时，不要强行二选一。先写成 `X+SLG`、`卡牌+Roguelike`、`经营+社交` 这类组合，再分别判断：

- `X` 层负责什么短期 Hook 和首日循环。
- `SLG` 或其他长线层负责什么长期 Link 和社交/竞争压力。
- 两层之间的资源、目标和 UI 是否接得上。

## 六个通用设计维度

| 维度 | 要看什么 | 常见风险 |
| --- | --- | --- |
| 核心循环 | 玩家动作 -> 判定 -> 资源/奖励 -> 变强 -> 下一目标是否闭合 | 只有奖励堆叠，没有下一轮燃料 |
| 成长曲线 | 战力、装备、等级、境界、图鉴、技能是否可读 | 数字涨了但玩家不知道为什么强 |
| 经济/福利 | 免费奖励、广告奖励、日常、离线收益、礼包是否分层 | 福利过密冲掉主目标 |
| 商业化打断 | 广告、首充、礼包、月卡、战令出现时机与当前痛点是否相关 | 玩家未理解循环前先被付费/广告拦截 |
| 反馈强度 | 数值、动画、音效、镜头、红点、爆装、结算是否让动作有回报 | 反馈不集中，无法形成爽点 |
| 长期目标 | 排行榜、开服目标、境界线、社交、公会、Boss、赛季是否接住当前行为 | 只有入口，没有可理解的追赶路线 |

## 系统叙事融合检查

当样本强调主题、社会系统、价值冲突、道德困境、涌现叙事或“玩法即叙事”时，启用这一组问题。否则不要为了显得深刻而强行展开。

| 控制点 | 要看什么 | 常见风险 |
| --- | --- | --- |
| 可玩命题 | 游戏想让玩家通过操作理解什么主题，而不是通过文案知道什么设定 | 主题只停留在剧情、旁白或美术氛围 |
| 价值轴系统 | 是否有互相对立的理念、派系、资源伦理或生活方式，并能被系统操作 | 阵营只是背景设定，不能影响玩法 |
| 玩家价值观权重 | 选择是否同时有数值代价和价值观代价 | 数学最优答案过强，道德困境变成假选择 |
| 后果具象化 | 抽象选择是否落到人物、城市、空间、秩序或长期状态变化 | 后果只在结算文本出现，没有可感对象 |
| 循环栈 | 短期掌握、战术选择、战略规划、长期社会后果是否层层递进 | 循环之间割裂，主题无法沉淀 |
| 文化对话循环 | 玩家离开后是否会讨论“这一切值得吗 / 我为什么这么选” | 通关后只剩数值复盘，没有意义复盘 |
| 意义注入 × 可能性空间 | 主题具体度与玩家可行动空间是否匹配 | 主题太重但玩法太窄，或自由太大但意义太散 |

判断时优先写成：

```text
主题命题 -> 可操作系统 -> 玩家动态 -> 情绪/意义 -> 后果证据 -> 验证指标
```

## 对截图的限制

截图能判断：

- 信息密度和主目标是否清晰
- UI 层级、入口拥挤、红点/弹窗压力
- 可见福利、广告、礼包、排行榜、长期目标
- 静态 Hook 和 Link

截图不能直接判断：

- 循环节奏
- 操作手感
- 等待时间
- 奖励是否真的推动下一轮
- 广告/付费是否实际打断流程

这些必须标 `uncertain`，或要求补录屏/视频链接。

## 对录屏/视频链接的最低证据点

至少尝试定位：

1. 首次可操作点。
2. 第一次核心动作。
3. 第一次奖励/资源变化。
4. 第一次变强反馈。
5. 第一次重复循环。
6. 第一次福利/广告/付费入口。
7. 第一次长期目标或排行榜/开服目标。
8. 第一个记忆峰值或可分享瞬间。

如果视频过长，先抽样：`0s / 30s / 60s / 3min / 7min / 15min / 25min / 后段`，再决定是否需要更密集抽帧。

## 建议输出

每条建议写成：

```text
证据定位 -> 设计判断 -> 修改动作 -> 预期效果 -> 验证指标
```

验证指标可以包括：

- time_to_first_action
- time_to_first_reward
- time_to_first_power_up
- repeated_core_action_count
- ad_prompt_count
- purchase_prompt_count
- popup_count
- red_dot_count
- next_day_goal_recall
- screenshot_shareability
