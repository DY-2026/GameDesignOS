# 单机游戏分析方法

这个文件用于用户点名单机、关卡、Boss、开放世界、叙事、解谜、动作冒险、通关体验，或样本明显是单机流程时。

## 1. VOI 门

先问：继续收集什么信息会改变判断？

| 信息 | 何时值得获取 | 不值得获取时怎么做 |
| --- | --- | --- |
| 当前版本、平台、补丁 | 影响难度、UI、关卡、商业化或内容量判断 | 只写样本版本 unknown，不做版本结论 |
| 外部评测/攻略/玩家反馈 | 需要判断通关结构、后期重复、关卡争议或 Boss 难度时 | 只分析当前片段，不推断全局口碑 |
| 竞品/同类案例 | 用户要求对标，或建议需要行业基准时 | 不泛搜竞品，只输出内部设计诊断 |
| 完整通关流程 | 用户要求整体单机或结局体验时 | 标记样本范围，只做首段/片段分析 |
| 设计理论来源 | 会改变分析框架或避免误用概念时 | 使用本文件默认框架 |

联网研究必须把“样本证据”和“外部资料”分开写。外部资料只能帮助解释和校准，不能替代当前截图、录屏或视频里的证据。

## 2. 样本范围门

单机游戏很容易被样本范围误导。先判定本次素材能支持哪一层结论：

| 样本范围 | 可判断 | 不可强判 |
| --- | --- | --- |
| 截图 | UI层级、目标可读性、空间/敌人/资源线索、静态氛围 | 操作手感、节奏、挑战曲线、谜题反馈 |
| 3-10 分钟片段 | 单段关卡、首次战斗、谜题、Boss阶段、短期节奏 | 整体内容量、后期重复、完整成长线 |
| 20-60 分钟录屏 | 首段流程、关卡节奏、学习曲线、功能引入 | 通关体验、后期难度、结局承接 |
| 多段或完整通关 | 整体结构、重复/变化、难度曲线、叙事承接、通关意愿 | 仍需区分玩家水平、剪辑和版本差异 |

输出时必须写清：本次是 `first_segment`、`level_slice`、`boss_slice`、`open_world_slice`、`narrative_slice`，还是 `full_run_review`。

## 3. 单机六问

### 3.1 Critical Path：主路径是否清楚

看玩家是否知道：

- 当前目标是什么。
- 下一步该去哪里。
- 为什么现在这个目标重要。
- 偏离主线是否有意义。
- 失败后从哪里重新进入。

证据：任务文案、地标、镜头引导、门/钥匙/敌人布置、地图标记、NPC对白、关卡出口、检查点。

### 3.2 Pacing Beats：节奏是否有变化

把片段拆成 beat：

- 学习：介绍动作、规则、敌人、线索。
- 测试：要求玩家用刚学到的内容做判断。
- 扭转：改变敌人组合、空间、资源或规则。
- 呼吸：探索、叙事、奖励、整理背包、观察环境。
- 峰值：Boss、setpiece、剧情揭示、稀有奖励。

风险信号：连续战斗无变化、连续文本无互动、连续跑图无选择、连续教学无验证。

### 3.3 Agency Map：玩家自主权来自哪里

不要把“能移动”误判为 agency。记录玩家真实能选择的东西：

- 操作选择：闪避、格挡、技能、走位、瞄准。
- 路线选择：主路、支路、绕行、潜行、探索。
- 解法选择：谜题路径、工具组合、环境利用。
- 风险选择：是否挑战强敌、是否消耗资源、是否回头补给。
- 叙事选择：对话、阵营、角色关系、道德代价。
- Build 选择：装备、能力、升级路线、消耗品。

判断标准：选择是否可理解、是否有代价、是否有反馈、是否改变后续体验。

### 3.4 Challenge-Skill-Feedback：挑战、技能、反馈是否闭合

检查玩家是否经历了：

1. 看懂目标。
2. 学到动作或规则。
3. 遇到略高于当前技能的挑战。
4. 通过反馈理解成功/失败原因。
5. 获得可复用的新能力、新知识或新路线。

常见断点：

- 敌人强，但失败原因不可读。
- 谜题难，但反馈只告诉“错了”。
- 操作多，但没有场景要求玩家区分使用。
- 奖励强，但和刚完成的挑战没有因果关系。

## 4. MDA 用在单机时怎么落地

| MDA 层级 | 单机分析要问 | 证据 |
| --- | --- | --- |
| Mechanics | 操作、敌人、谜题、关卡规则、资源、检查点、能力、路线约束是什么？ | UI、按键提示、战斗规则、关卡门槛、资源消耗 |
| Dynamics | 玩家在这些规则下形成什么行为？谨慎、探索、刷资源、逃避、试错、精通、收集？ | 操作选择、失败重试、路径选择、奖励追逐、战斗节奏 |
| Aesthetics | 最终体验是什么？紧张、孤独、英勇、好奇、掌控、惊奇、恐惧、治愈、成就？ | 视听表现、节奏峰值、叙事承诺、玩家目标变化 |

重点找断裂：

- 机制承诺动作爽感，但动态是站桩数值交换。
- 叙事承诺自由选择，但机制只允许一本道。
- 开放世界承诺探索，但奖励结构只鼓励清图标。
- 恐怖承诺脆弱，但资源过量让玩家无压力。

## 5. 类型分支策略

### 线性动作 / 动作冒险

重点看：

- teach / test / twist 是否存在。
- 新能力是否被关卡立刻验证。
- 检查点是否减少挫败而不消灭张力。
- Boss 阶段变化是否逼出新判断。
- setpiece 是否有可玩性，而不是纯观看。

### 开放世界 / 半开放探索

重点看：

- 地标是否比图标更先吸引玩家。
- 主线、支线、探索、收集是否有清晰优先级。
- 绕路是否产生发现、风险或故事，而不是空跑。
- 地图密度是否让玩家选择路线，而不是清单劳动。
- 探索奖励是否回到核心成长或世界理解。

### 叙事 / 解谜 / 冒险

重点看：

- 悬念是否让玩家主动寻找线索。
- 线索是否可见、可解释、可验证。
- 谜题反馈是否帮助玩家修正假设。
- 文本、演出和互动比例是否合理。
- 选择是否有机制、情绪或关系后果。

### 生存 / 沉浸模拟 / 资源管理单机

重点看：

- 稀缺是否真实约束行为。
- 风险是否能被侦查、计划和承担。
- 系统是否允许多种解法。
- 失败是否生成故事，而不是只惩罚时间。
- 资源、噪音、视野、敌人 AI 是否共同制造动态。

## 6. 报告必须包含的单机字段

| 字段 | 要求 |
| --- | --- |
| `single_player_scope` | `first_segment` / `level_slice` / `boss_slice` / `open_world_slice` / `narrative_slice` / `full_run_review` |
| `critical_path` | 主路径证据、偏离空间、路径风险 |
| `pacing_beats` | beat 列表，至少写学习/测试/变化/呼吸/峰值中出现的部分 |
| `agency_map` | 玩家真实选择、代价、反馈、后果 |
| `challenge_skill_feedback` | 目标、挑战、技能要求、成功/失败反馈 |
| `narrative_mechanic_fit` | 叙事承诺和机制动态是否互相支撑 |
| `finish_intent` | 这段样本如何制造继续推进、探索、精通或通关意愿 |

## 7. 外部来源校准

这些来源用于校准框架，不是替代样本观察：

- MDA 框架：Mechanics、Dynamics、Aesthetics 三层帮助把机制、玩家行为和体验目标连起来。来源：<https://www.cs.northwestern.edu/~hunicke/MDA.pdf>
- Playcentric design：设计判断应回到原型、试玩和迭代，而不是只停留在理论解释。来源：<https://www.gamedesignworkshop.com/>
- Level Design Book：critical path 和 pacing 可用于拆单机关卡、开放世界路线和节奏 beat。来源：<https://book.leveldesignbook.com/process/layout/criticalpath>、<https://book.leveldesignbook.com/process/preproduction/pacing>
- GameFlow：挑战与技能匹配、清晰目标和反馈是可玩体验的重要检查点。来源：<https://www.valuesatplay.org/wp-content/uploads/2007/09/sweetser.pdf>
- Microsoft Xbox Accessibility Guidelines：镜头晃动、动态背景、闪烁、自动更新 UI 等会影响可读性和舒适度。来源：<https://learn.microsoft.com/en-us/gaming/accessibility/xbox-accessibility-guidelines/117>
