# 前瞻机会判断镜头

这个镜头用于用户问“这个方向值不值得做 / 还有没有窗口 / 能不能火 / 这个题材或玩法能不能迁移 / 大厂做了我们还能不能跟”。它可以和 PV/宣传片热度预测、品类对标、整体分析或玩法机制分析组合使用。

## 1. 边界

前瞻判断不是预测未来，而是判断：

- 当前样本展示的机会是什么。
- 这个机会解决了旧品类或旧表达的什么痛点。
- 市场、平台、用户情绪和竞品环境是否给它窗口。
- 现在最值得做的低成本验证是什么。
- 什么信号出现就应该停止投入。

不要把“题材吸量”直接等同于“产品长期成立”。题材、玩法、商业化、平台分发和研发供给必须分别判断。

## 2. 三路信号

默认用三路信号判断前瞻机会：

| 信号 | 要问什么 | 常见证据 |
| --- | --- | --- |
| 创新源头 | 这个机制、题材表达、叙事结构或投放表达从哪里来？解决了旧模型什么痛点？ | Steam/主机/独立游戏、桌游、模组、短视频内容、相邻品类、老品类疲劳点 |
| 市场验证 | 有没有已有样本证明用户愿意看、愿意玩、愿意付费或愿意传播？ | 播放/留存/评论、预约/愿望单、榜单、测试反馈、买量素材、社区讨论 |
| 时代情绪 | 玩家现在在厌倦什么，又在奖励什么？ | 反重肝、反重氪、反重复、低理解成本、强主题、先爽后深、反上班、强身份表达 |

权重不是固定的。PV 分析更重视市场验证和时代情绪；玩法迁移更重视创新源头和结构适配；立项判断三者都要看。

## 3. 机会类型

每次前瞻判断必须先标注机会类型：

| 类型 | 含义 | 常见风险 |
| --- | --- | --- |
| 结构创新 | 核心循环、社交结构、单局结构、资源结构真的变了 | 学习成本高，研发成本高，验证周期长 |
| 内容创新 | 题材、美术、叙事、角色、世界观供给强 | 吸量但不一定留存 |
| 题材创新 | 旧玩法换上新的文化、身份或情绪入口 | 容易变成皮相包装 |
| 投放创新 | 素材表达、首秒冲突、短视频结构更强 | 买量强但产品承接弱 |
| 组合创新 | 把 X 层短循环与 Y 层长线结构重新组合 | 系统边界不清，用户预期错配 |
| 反转窗口 | 旧头部疲劳，新用户需求出现，但新原型未被验证 | 看见机会早，但商业模型未闭合 |

## 4. 时间窗口

统一按以下阶段判断：

| 阶段 | 判断 | 行动 |
| --- | --- | --- |
| 源头萌芽期 | 只有少数源头样本，市场尚未验证 | 做概念原型和素材测试 |
| 跟进窗口期 | 1-2 个强样本开始验证 | 快速做差异化原型和用户测试 |
| 验证窗口期 | 已有成功案例，但仍有细分空间 | 选择细分受众或强差异点 fast-follow |
| 红海期 | 多个成功产品和强投放进入 | 除非有护城河，否则谨慎 |
| 反转窗口期 | 老模型疲劳，玩家反感点明显 | 用低成本样本验证新入口 |

默认经验：

- 休闲轻度：默认只看 1-3 个月验证窗口。超过 3 个月仍没有素材、原型或转化信号，优先降级为 Watch 或 No-Go。
- 微小中重度：默认只看 3-6 个月验证窗口。超过 6 个月仍不能证明核心玩法、留存或付费承接，优先停止扩大投入。
- 大体量 PC/主机或长期研发项目可以另算研发周期，但前瞻机会判断仍先按上述窗口要求产出阶段性证据。

这些只是默认估计，报告必须把当前证据和假设写清楚。

输出时必须区分：

- `remaining_window_estimate`：机会还能观察/验证多久。
- `minimum_validation_deadline`：下一轮最低成本验证最晚什么时候交付。
- `production_timeline`：完整研发周期；不要用它替代窗口期判断。

## 5. 品类适配检查

任何迁移或机会判断，都要先写：

1. 源环境为什么成立：用户动机、会话时长、输入方式、经济模型、内容供给。
2. 目标环境有什么不同：平台、用户期待、社交形式、商业化、版本节奏。
3. 可迁移：结构、反馈、情绪价值、题材表达。
4. 不可迁移：输入密度、单局张力、内容成本、社区关系、付费逻辑。
5. 需要的中间层：教程、自动化、轻量化、长线目标、社交承接、素材重剪。

必须给一句明确判断：

```text
这是结构可迁移、题材可迁移、表达可迁移，还是暂时只适合做营销测试。
```

## 6. Go / No-Go

前瞻结论必须落到以下之一：

| 结论 | 含义 |
| --- | --- |
| Go | 样本、窗口、用户情绪和验证路径都较清楚，可以推进下一阶段 |
| Conditional Go | 有机会，但必须先通过一个最小验证 |
| No-Go | 结构、窗口或转化证据不足，不建议投入 |
| Watch | 暂不做，持续观察某些信号 |

## 7. 最小验证

每个建议都要写验证对象和 Kill 条件。

| 验证对象 | 可用方法 | 关键指标 | Kill 条件 |
| --- | --- | --- | --- |
| 题材吸量 | PV/封面/标题 A/B，小额投放，站内动态测试 | CTR、3秒留存、评论情绪、收藏转发 | 只有好奇，没有复述卖点 |
| 核心玩法 | 可玩 Demo、灰盒原型、小样本试玩 | 复玩意愿、失败复盘、操作理解、推荐意愿 | 玩家只夸题材，不想再玩 |
| 商业化承接 | 假门面礼包/预约/愿望单/付费意愿访谈 | 点击、预约、愿望单、付费意愿 | 热度无法转到任何行动 |
| 长线结构 | 纸面经济模型、D1/D7 目标推演、赛季草模 | 回访理由、目标清晰度、系统容量 | 首日后没有追求对象 |
| 跨品类迁移 | 源机制拆解 + 目标品类低保真原型 | 是否保留原机制爽点 | 迁移后只剩皮相 |

## 8. 报告字段

| 字段 | 要求 |
| --- | --- |
| `opportunity_type` | 结构创新 / 内容创新 / 题材创新 / 投放创新 / 组合创新 / 反转窗口 |
| `source_signal` | 创新源头和解决的旧痛点 |
| `market_validation_signal` | 已有播放、互动、榜单、预约、测试、评论等证据 |
| `era_emotion_signal` | 用户当前厌倦和奖励的体验 |
| `window_stage` | 源头萌芽 / 跟进窗口 / 验证窗口 / 红海 / 反转窗口 |
| `remaining_window_estimate` | 休闲轻度默认 1-3 个月；微小中重度默认 3-6 个月；证据不足时写假设 |
| `minimum_validation_deadline` | 下一轮最低成本验证截止时间，不等同于完整研发周期 |
| `transferability` | 结构可迁移 / 题材可迁移 / 表达可迁移 / 只适合营销测试 |
| `go_no_go` | Go / Conditional Go / No-Go / Watch |
| `minimum_validation` | 下一步最低成本验证 |
| `kill_conditions` | 停止投入条件 |
