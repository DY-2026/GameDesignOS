# Visual Evidence Card / 关键截图解释卡

用于把关键截图从“插图”升级为“证据”。每张关键截图都必须解释它支撑了什么判断、不能支撑什么判断、下一步如何迭代。

## 使用规则

- 每张关键截图必须绑定 `evidence_id`，并能在 Evidence Index 中找到同一 ID。
- 截图下方必须写可观察事实，不允许只写“这里很好/这里不好”。
- 截图解释必须包含设计含义和迭代动作；否则只是配图，不是诊断证据。
- 如果截图只能支撑静态判断，必须明确不能判断手感、节奏或循环闭合。

## 字段定义

| 字段 | 必填 | 规则 |
| --- | --- | --- |
| `evidence_id` | 是 | 对应 Evidence Index。 |
| `image_path` | 是 | 相对路径或报告可访问路径。 |
| `caption` | 是 | 用一句话说明截图在诊断里的角色。 |
| `observed_fact` | 是 | 只写画面中能看见的事实。 |
| `design_meaning` | 是 | 解释这个事实为什么影响体验、转化或验证。 |
| `judgment` | 是 | 可执行诊断判断，必须受样本边界约束。 |
| `iteration_action` | 是 | 下一步改法，不能写空泛优化。 |
| `confidence` | 是 | `high` / `medium` / `low` / `unsupported`。 |

## 模板

```markdown
### E001 截图标题：一句话诊断

![E001 截图标题](./path/to/image.jpg)

| 解释项 | 内容 |

| 设计含义 |  |
| 诊断判断 |  |
| 迭代动作 |  |
| 置信度 | high / medium / low / unsupported |
```

## 最小示例

```markdown
### E001 首屏福利弹窗：Hook 被福利层抢占

![E001 首屏福利弹窗](./survival-33-days-assets/survival33-00-00-welfare-popup.jpg)

| 解释项 | 内容 |
| --- | --- |
| 可观察事实 | 进入第一眼是全屏福利活动：“888代金券”“前往参与”。 |
| 设计含义 | 玩家先理解福利入口，而不是丧尸生存危机。 |
| 诊断判断 | 首屏 Hook 顺序错误；福利应该服务留存承接，不应替代题材钩子。 |
| 迭代动作 | 先给 3-5 秒危机场面，再在首战后弹福利。 |
| 置信度 | high |
```
