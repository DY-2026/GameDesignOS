# 00 Inbox

Use this directory for raw, unreviewed intake:

- rough ideas and meeting fragments;
- links, screenshots, and competitor observations;
- questions and proposed research;
- AI outputs that have not been mapped to a project decision.

Inbox material is not design truth and not automatically a research backlog. Before promotion, classify each useful item:

```text
decision_information   -> may change a named current action
model_learning         -> improves a reusable model but does not change the current action
information_consumption -> entertainment, identity, curiosity, or FOMO relief
```

Promote decision information only after its source, target decision, action-change mechanism, and review boundary are clear. Give model learning a budget and reuse target. Consumption may be kept intentionally, but it must not occupy the project decision thread.

Do not let an agent treat the newest inbox note as an accepted decision or open a new workflow branch without an owner, exit, and stop rule.
