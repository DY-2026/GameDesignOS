# Showcase Index

This index points new users to the smallest public-facing proof paths for `ParanoiaSkills`. Showcase material must stay synthetic, public, explicitly cleared, or marked `needs_review` until cleared.

## Game Experience Analyzer

| Field | Value |
| --- | --- |
| target user | Game designers, product reviewers, indie teams, and agent users who need media-based game experience diagnosis. |
| input material | PV, trailer, gameplay recording, screenshot set, or video link. |
| skill used | `$game-experience-analyzer` |
| output artifact | Evidence-linked diagnosis report with sample boundary, evidence index, diagnosis route, issue cards, and validation recommendations. |
| what this proves | A media sample can become a reviewable diagnosis without pretending to know retention, revenue, or private telemetry. |
| link to example or placeholder issue | [Survival 33 Days gameplay experience report](../../game-experience-analyzer/examples/survival-33-days-gameplay-experience-report.md) |

## Game Concept Architect

| Field | Value |
| --- | --- |
| target user | Indie developers, designers, and agent users turning rough ideas into testable concept plans. |
| input material | One-line game idea or rough concept seed. |
| skill used | `$game-concept-architect` |
| output artifact | Verifiable design blueprint with concept seed, assumption ledger, player promise, core loop, scope gate, and validation plan. |
| what this proves | A one-line idea can become a bounded design plan without becoming an oversized GDD. |
| link to example or placeholder issue | [Clockwork Garden Defense synthetic example](../../game-concept-architect/examples/clockwork-garden-defense.md) |

## Concept-to-Diagnosis Loop

| Field | Value |
| --- | --- |
| target user | Teams that want to move from player promise to prototype/media review using a consistent evidence contract. |
| input material | Player promise contract, prototype recording, screenshot set, PV, or synthetic test sample. |
| skill used | `$game-concept-architect` followed by `$game-experience-analyzer` |
| output artifact | Player promise contract, prototype/media diagnosis, unsupported-claim list, and next validation plan. |
| what this proves | The same promise can drive both concept architecture and later evidence-linked diagnosis. |
