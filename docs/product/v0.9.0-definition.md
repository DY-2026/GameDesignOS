# v0.9.0 — Local Runtime Prototype

## Release Theme

From documented runtime semantics to an executable local control plane.

v0.8.0 proved the workspace, contract, VOI, and workflow structure. v0.9.0 must prove that those structures can be operated consistently by deterministic commands instead of manual copying and ad hoc file edits.

## Product Promise

A designer can initialize a workspace, see its current state, define an information decision, receive an explainable skill route, create the next durable asset, validate references and boundaries, and produce a review-safe snapshot without sending project material to a hosted service.

## Shipped Capabilities

```text
init
status
voi
route
new
validate
pack
doctor
```

The CLI is installable through `pyproject.toml` and also runs through `python -m gamedesignos`.

## Success Criteria

v0.9.0 succeeds when:

1. a private and a public-synthetic workspace can be initialized from scratch;
2. the runtime records workspace schema and runtime version separately;
3. an ED or proposal request with missing upstream assets is routed to the smallest prerequisite;
4. a VOI assessment cannot contain more than three candidate information actions;
5. signal-to-action mappings that do not change action are recognized as low-value research;
6. draft asset creation never invents evidence or marks an asset accepted;
7. workspace validation catches missing files, unsafe paths, broken asset references, and public/private conflicts;
8. public packs use a registered-asset allowlist and stop on ambiguous source status;
9. at least two synthetic projects are covered by automated tests;
10. the runtime neither stores credentials nor calls a model.

## Non-Goals

- hosted API or SaaS;
- model-provider integration;
- automatic skill execution;
- autonomous search or RAG;
- exact monetary VOI calculation from incomplete inputs;
- automatic project migration beyond compatible v0.8 manifests;
- GUI dashboard;
- game-engine adapter;
- team permissions;
- replacing Human Gates.

## Runtime Authority

The CLI may create drafts, indexes, reports, and packs. It may not accept, reject, publish, fund, scope, release, or reverse a project commitment. Those remain human actions recorded in the decision log.

## Compatibility Contract

The workspace schema stays at `0.8.0` because the manifest shape has not changed. `designos.version` becomes `0.9.0`. The project-workspace schema accepts both v0.8 and v0.9 runtime declarations. Future schema changes require an explicit migration command and fixture coverage.

## Exit Gate

Before promotion:

```text
python scripts/validate_repo.py
python -m unittest discover -s scripts/tests
python -m pip install -e .
gamedesignos --version
gamedesignos doctor
```

The release remains a prototype until CI passes and a maintainer reviews the generated files, routing failures, packaging exclusions, and rollback behavior.
