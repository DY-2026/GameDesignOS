# GameDesignOS v1.0 开发计划：Project-Ready GameDesignOS

> 状态：v1.0.0 已实现，等待 maintainer release / 真实项目验证  
> 日期：2026-06-19  
> 目标版本：v1.0.0 Project-Ready GameDesignOS

## 1. 一句话定位

GameDesignOS v1.0 不是“大而全版本”，而是一个本地优先、决策驱动、证据约束、AI 可协作的游戏设计操作系统。

它要回答的问题是：

```text
一个设计决定是怎么产生的？
它基于什么证据？
验证过什么？
谁拍板？
结果如何？
以后还能不能复用？
```

## 2. WOOP Task Card

### Wish / Intent Spec

把 v0.9 的 Local Runtime Prototype 升级为 Project-Ready 的 v1.0 路线，使真实游戏项目可以连续使用 GameDesignOS 管理至少一个完整周期：

```text
想法 -> 假设 -> 证据 -> 实验 -> 决策 -> 复盘 -> 下一轮演化
```

本计划的执行边界：

- 不把 v1.0 理解成 GUI、云协作、多用户权限或复杂多 agent。
- 不默认迁移旧 workspace。
- 不让 agent 自动 accept decision、发布承诺、删除资产或锁定范围。
- 先完成本地 Project-Ready 主路径，再把 GUI、云协作、多用户权限等留到后续版本。

### Outcome / Evaluation Rubric

v1.0 完成后，一个真实项目至少应能被复查出：

1. 哪些 Decision Object 产生过，当前状态是什么。
2. 哪些 Assumption 未验证、已验证或已证伪。
3. 哪些 Evidence 支撑过决策，以及不能证明什么。
4. 哪些 Experiment 产生了结果和复盘。
5. 哪些 Learning 可以复用，哪些仍只是 candidate。
6. 哪些高影响承诺被 Human Gate 或 Rollback Gate 阻断过。

### Obstacle / Failure Pattern

最可能破坏 v1.0 的不是“功能不够”，而是以下失败模式：

- 把更多命令误认为更强 OS。
- 把漂亮文档误认为可运行的项目记忆。
- 把 AI 建议误认为 Human Gate 决策。
- 把一次实验结果直接提升成长期规则。
- 在目录重构时破坏 v0.9 已可用 CLI。

### Plan / If-Then Protocol

- 如果新功能无法用行为测试证明，就保持 `candidate`。
- 如果会破坏 v0.9 workspace，就改为兼容新增，不做强制迁移。
- 如果动作会进入承诺态、发布态、删除态或全局安装态，就停在 Human Gate。
- 如果 schema、CLI 和文档不一致，就以 schema 与测试为准，文档回改。

## 3. v1.0 核心原则

1. **Decision First**：所有项目推进都应能追到 Decision Object。
2. **Evidence Bound**：强判断必须标出证据边界和 unsupported claims。
3. **Experiment Before Commitment**：高影响、低可逆、高投入的方向必须先实验或说明跳过理由。
4. **Human Gate Owns Commitment**：AI 只能建议、检查、阻断，不能替人接受承诺。
5. **Learning Must Persist**：复盘后的学习先进入 candidate，再经 eval 和 Human Gate 提升。

## 4. 六层架构

```text
L6 Interface Layer
L5 Intelligence Layer
L4 Workflow Orchestration Layer
L3 Decision & Gate Layer
L2 Contract & Memory Layer
L1 Runtime Kernel
```

v1.0 的实现优先级从 L2/L3 开始：先把 Decision、Assumption、Evidence、Experiment、Learning、GateResult 做成结构化对象，再让 workflow 和 intelligence 层读取这些对象。

## 5. 5 个 PR / Milestone

### PR 1：Core Models + Workspace Contract

交付：

- `decision.schema.json`
- `assumption-registry.schema.json`
- `evidence-ledger.schema.json`
- `experiment-plan.schema.json`
- `experiment-result.schema.json`
- `learning-record.schema.json`
- `gate-result.schema.json`
- v1 workspace template
- runtime 可读取 v1 记录的 Project-Ready state

验收：

- 对象能 create / validate / serialize / load。
- v0.9 workspace 不被破坏。
- 行为测试覆盖高风险无 rollback、near decision 无 VOI、未验证假设等场景。

### PR 2：Decision Graph

交付：

- `graph inspect`
- `graph export --format mermaid`
- 支持 Decision、Assumption、Evidence、Experiment、Learning 的最小边类型。

验收：

- 能从一个 decision 追到 assumption、evidence、experiment、learning。
- Mermaid 输出可读，并可用于文档复查。

### PR 3：Gate System

交付：

- `gate run voi`
- `gate run evidence`
- `gate run scope`
- `gate run experiment`
- `gate run commitment`
- `gate run rollback`

验收：

- 无 Decision Object 时，VOI Gate 阻止 research。
- 高影响 decision 无 rollback 时，Commitment/Rollback Gate 阻断。
- linked experiment 未 review 时，decision 不可进入承诺态。

### PR 4：Workflow Engine

交付：

- `workflow list`
- `workflow start`
- `workflow status`
- `workflow next`
- `workflow validate`

验收：

- `idea_to_validated_prototype` 能跑完整链路。
- 系统能指出“卡在哪一步、缺哪个资产、下一步最小动作是什么”。

### PR 5：Project Health + Pack

交付：

- `health`
- `next`
- `pack internal-review`
- `pack publisher`
- `pack public-synthetic`

验收：

- synthetic project 可输出健康扫描。
- public-synthetic pack 不导出 private evidence。
- rejected/superseded 下游资产可被标记 stale。

## 6. v1.0 最小可发布范围

必须包含：

- v1 workspace structure
- core models
- decision graph
- assumption registry
- evidence ledger
- experiment plan/result/review
- gate system
- workflow engine
- health scan
- Mermaid graph export

暂不包含：

- GUI
- 在线协作
- 账号权限
- 云同步
- 自动模型调用
- 游戏引擎插件
- 复杂可视化
- 多用户审计

## 7. v1.0.0 已实现范围

v1.0.0 已完成最小可发布范围：

- 新增 Project-Ready runtime 模块，读取 Decision、Assumption、Evidence、Experiment、Learning。
- 新增 `gamedesignos ask` 和 `gamedesignos "<一句话>"`，让用户拉取项目后可以直接用自然语言进入路由或项目初始化。
- 新增 `gamedesignos start` 一键创建或恢复 v1 workspace、第一条 Decision、Assumption、Experiment、VOI Gate 和 Workflow Run。
- 默认 `gamedesignos init` 创建空 v1 workspace，并保留 `--workspace-version 0.8.0` legacy 模式。
- 新增 `decision new/list/inspect/accept/reject/supersede`。
- 新增 `assumption new/list/validate`。
- 新增 `evidence add/list/inspect`。
- 新增 `experiment plan/result/review`。
- 新增 `gate run`、`health`、`next`、`graph export/inspect`。
- 新增 `workflow list/start/status/next/validate`。
- 新增 v1 contract schemas、v1 workspace template、中文 runtime/CLI/contracts/roadmap/release docs。

这些能力已经通过 synthetic runtime 行为测试；真实私有项目仍应作为 release 后验证门，用来判断后续 v1.1/v1.2 是否需要 GUI、引擎 adapter 或团队协作层。

## 8. Rollback

v1.0 rollback 边界清晰：

- 删除 `gamedesignos/project_ready.py`。
- 从 `gamedesignos/cli.py` 移除 `health`、`next`、`gate`、`graph` 入口。
- 删除新增 v1 schemas、v1 workspace template 和测试文件。
- 保留 v0.9 runtime、workspace 和现有 skill 不变。

不需要移动项目根目录，不需要替换 `.git`，不需要强制迁移任何旧 workspace。
