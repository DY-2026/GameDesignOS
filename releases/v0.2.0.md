# v0.2.0 - Concept-to-Diagnosis Skill Loop

Release title: **Concept-to-Diagnosis Skill Loop**

`ParanoiaSkills` v0.2.0 upgrades the public repository from a method preview into a five-skill agent library that can be installed, validated, and safely cited.

## What Changed

- Added `game-concept-architect`.
- Clarified the public skill architecture across design production, workflow governance, and knowledge asset layers.
- Added public example boundaries so examples use only synthetic, public, or explicitly cleared cases.
- Added adapter guidance for Markdown-skill hosts and OpenAI-compatible harnesses.
- Added repository and per-skill validation scripts.

## Public Loop

The current public loop is:

```text
one-line idea
-> verifiable design blueprint
-> prototype/media
-> evidence-linked diagnosis
```

`game-concept-architect` turns a one-line idea into a design blueprint with seed extraction, player promise, core loop, scope gate, production feasibility, and validation plan. After a prototype, trailer, recording, screenshot set, or store page exists, `game-experience-analyzer` turns the material into evidence-linked diagnosis and validation-ready fixes.

## Installation

Copy these five skill folders into an agent environment that can load Markdown skills:

```text
ParanoiaSkills/game-concept-architect -> your-agent-skills/game-concept-architect
ParanoiaSkills/game-experience-analyzer -> your-agent-skills/game-experience-analyzer
ParanoiaSkills/paranoia-ai-system-evolver -> your-agent-skills/paranoia-ai-system-evolver
ParanoiaSkills/game-design-book-translator -> your-agent-skills/game-design-book-translator
ParanoiaSkills/game-design-source-curator -> your-agent-skills/game-design-source-curator
```

Then validate the repository:

```text
python scripts/validate_repo.py
```
