# v0.4.0 - Experience Concentration Skill

This is a Markdown release draft only. Do not create the GitHub release or Git tag until the final release commit is reviewed and validated.

## What Changed

- Added `game-experience-density-optimizer`, a new public skill for turning experience concentration problems into weekly ED experiments.
- Standardized the Chinese concept name as `体验浓度`, with `ED / Experience Density` as the technical shorthand.
- Added the working formula `ED = MD/min * (SF + EB + AR) / CLP` as a design heuristic, not a scientific claim.
- Added a diagnosis order: reduce cognitive noise, improve vertical quality, then tune meaningful-decision frequency.
- Added templates for intake, weekly ED plans, variant matrices, instrumentation dictionaries, dashboard specs, weekly reviews, and structured output.
- Added evals, negative cases, and a synthetic public example for first-session ED planning.
- Added generated overview/showcase images and updated the README showcase layout to four cards per row.
- Updated repository validation so `game-experience-density-optimizer` is part of the required public skill set.

## Why This Release Matters

`ParanoiaSkills` now covers the path from evidence diagnosis to concrete tuning experiments. The new skill helps turn vague retention, pacing, feedback, embodiment, atmosphere, and cognitive-load complaints into rollbackable weekly A/B plans.

It explicitly keeps the method honest: `体验浓度` is a design hypothesis and practical lens, not a validated psychological scale.

## Try It

```text
Use $game-experience-density-optimizer to turn this first-session retention, pacing, or experience concentration problem into an ED diagnosis, CLP/SF/EB/AR/MD-min levers, a weekly A/B plan, instrumentation, dashboard fields, decision rules, and rollback gates.
```
