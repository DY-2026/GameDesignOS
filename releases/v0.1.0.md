# v0.1.0 - Game Experience Analyzer Preview

Release title: **Game Experience Analyzer Preview**

This is a historical preview release and does not list newer skills.

`ParanoiaSkills` turns game design analysis, AI workflow evolution, translation, and source curation into reusable agent skills. This preview release centers the most immediately useful package: `game-experience-analyzer`.

## Installation

Copy one or more skill folders into an agent environment that can load Markdown skills:

```text
ParanoiaSkills/game-experience-analyzer -> your-agent-skills/game-experience-analyzer
ParanoiaSkills/paranoia-ai-system-evolver -> your-agent-skills/paranoia-ai-system-evolver
ParanoiaSkills/game-design-book-translator -> your-agent-skills/game-design-book-translator
ParanoiaSkills/game-design-source-curator -> your-agent-skills/game-design-source-curator
```

Then call the skill by name:

```text
Use $game-experience-analyzer to analyze this gameplay recording into timestamped evidence, feature exposure/unlock/first-use ledger, Hook/Loop/Link/Surprise diagnosis, and actionable fixes.
```

## Supported Inputs

- Screenshots: UI hierarchy, static hook, visible rewards, entry-point crowding, and UX risk.
- Local gameplay recordings: timestamped timeline, event stream, feature ledger, onboarding, loop, pacing, and issue priority.
- Trailers / PVs: heat-potential tier, selling-point clarity, proof-of-play boundary, platform fit, and validation plan.
- Video links: access notes, visible evidence, page metadata, and degraded analysis when the video cannot be fully inspected.

## Known Limitations

- A screenshot cannot prove pacing, feel, wait time, or loop closure without dynamic evidence.
- A PV can predict heat potential and validation path, but it cannot prove full gameplay quality, retention, sales, revenue, or long-term product quality.
- Video-link analysis depends on access permissions, region, login state, platform visibility, and available extraction tools.
- The skill defaults to Chinese reports because its current examples, templates, and evaluation prompts are optimized for Chinese game-design writing.

## Preview Cases

1. **Survival 33 Days gameplay recording review**
   A public 41-minute gameplay recording becomes a report with visual evidence, event timeline, feature ledger, early-loop diagnosis, UI/guide risks, and validation-ready fixes.

2. **Screenshot UX / static hook audit**
   A small screenshot set can produce a visual evidence table, UI hierarchy diagnosis, uncertain dynamic claims, and a list of follow-up clips needed.

3. **Gameplay recording experience review**
   A local recording can become a timestamped event stream, Hook/Loop/Link/Surprise scores, feature exposure/unlock/first-use ledger, and owner/action/validation recommendations.
