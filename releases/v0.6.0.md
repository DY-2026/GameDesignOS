# v0.6.0 - Proposal Assembly and ED Experiment Compiler

This is a Markdown release draft only. Do not create the GitHub release or Git tag until the final release commit is reviewed and validated.

## Highlights

- Adds `game-design-proposal-writer` as the seventh public skill package, turning concept briefs, evidence notes, validation plans, source notes, and production constraints into reviewable proposals, pitch outlines, decision memos, and vertical-slice documents.
- Upgrades `game-experience-density-optimizer` into an evidence-gated experiment compiler with explicit output modes, weekly A/B plans, scorecards, instrumentation, dashboard fields, decision rules, and rollback gates.
- Refreshes the GitHub landing page for the seven-skill architecture with a new generated banner, a clearer one-sentence project intro, and updated About copy.

### Added

- New skill package: `game-design-proposal-writer/`.
- Proposal-writing references, templates, examples, evals, negative cases, and synthetic output fixtures.
- ED evidence gate: `game-experience-density-optimizer/references/evidence-gate.zh-CN.md`.
- ED scorecard, expanded experiment schemas, weekly experiment plan updates, and synthetic public examples for premium-demo and hybrid-conflict review scenarios.
- New banner asset: `assets/paranoia-skills-overview-banner-v6.png`.

### Changed

- Updated `README.md`, `README.zh-CN.md`, and `README.en.md` for `v0.6.0`, seven skills, proposal writing, and the new banner.
- Updated `docs/GITHUB_ABOUT.md` and `docs/github-about-checklist.md` around the verifiable game-design agent-skill positioning.
- Updated `docs/showcases/README.md` with proposal assembly and the current ED compiler path.
- Updated `contracts/router.yaml` and `contracts/README.md` so routing and handoff language match the new proposal/ED flow.
- Updated `scripts/validate_repo.py` so the new skill is part of repository validation.

### Safety

- Public examples must remain synthetic, public, cleared, or marked `needs_review`.
- Real projects, client work, private screenshots, private recordings, local paths, and private overlays remain outside the public repository.
- The banner background is generated without readable text; exact project text and version were overlaid locally to avoid AI-generated typo artifacts.
