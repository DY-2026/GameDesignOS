# GameDesignOS v1.1.0 — RJR-AI Authority Layer

## Highlights

v1.1.0 upgrades GameDesignOS from “Project-Ready local runtime” to a clearer AI engineering system with an explicit residual-judgment boundary.

The public one-line GitHub description is now:

> Local-first AI game design OS for decisions, evidence, experiments, proposals, workflow authority, and human-gated project memory.

The core RJR-AI rule:

```text
AI expands possibilities.
Workflow compresses chaos.
Eval provides feedback.
Permission gates prevent overreach.
Knowledge bases preserve organizational memory.
Humans retain the residual judgment for high-coupling, low-reversibility, under-evidenced choices.
```

## Added

- RJR-AI guidance in `paranoia-ai-system-evolver`.
- RJR-AI eval coverage for AI/workflow/eval/permission/knowledge-base responsibility splits.
- GitHub About metadata guidance with the updated description and `ai-workflows` topic.
- Release-level documentation for the runtime/package version bump.

## Changed

- Runtime/package version is now `1.1.0`.
- Project-Ready workspace schema remains `1.0.0`.
- README trio now presents GameDesignOS as a local-first AI game design OS with workflow authority and human-gated project memory.
- Runtime docs, product vision, roadmap, changelog, package metadata, and v1 manifest are synced to v1.1.
- Router language now treats RJR-AI and residual-judgment requests as `paranoia-ai-system-evolver` work.

## Safety

- RJR-AI does not let AI replace leadership judgment.
- Human Gate remains required for commitment-changing, high-risk, or irreversible choices.
- Permission boundaries remain part of the system design, not optional prose.
- No existing v1 workspace migration is required.
- Installed runtime skill copies are not mutated automatically by this source-repo release.

Recommended checks:

## Rollback

If v1.1 must roll back:

1. Keep v1 workspaces in place; `schema_version` did not change.
2. Revert `pyproject.toml` and `gamedesignos.constants.RUNTIME_VERSION` to `1.0.0`.
3. Revert README/GitHub About wording to the v1.0 Project-Ready positioning.
4. Keep RJR-AI changes as `candidate` notes if they need more real-project validation.
