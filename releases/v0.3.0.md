# v0.3.0 - Game Dissection Upgrade

Release title: **Game Dissection Upgrade**

`ParanoiaSkills` v0.3.0 重构了“概念设计 -> 经验诊断”的闭环：从“为什么好玩”扩展为“机制迁移判断 + 证据链 + 验证路径”。

## Highlights

- 补齐 `game-concept-architect` 的可迁移分析链路（动词-对齐-不确定性-系统动态-内容流）。
- `game-experience-analyzer` 新增完整游戏拆解与机制迁移视角的报告路径。
- 引入更严格的评估与负向样例约束，降低“截图直觉输出”和“竞品表层复刻”风险。
- 用于发版的说明模板同步落地到 `CHANGELOG.md` 与 `releases/README.md`。

## Added

- `game-concept-architect` 新增与拆解链路一致的结构化输出：
  - `player-verb-inventory`
  - `action-goal-alignment`
  - `uncertainty-calibration`
  - `system-dynamics-map`
  - `content-flow-plan`
  - `audience-desire-map`
  - `playable-theme-map`
- `game-experience-analyzer` 新增 `game_dissection_diagnosis` 参考输出与模板约束。
- 仓库新增 `CHANGELOG.md`，集中记录版本变更记录。
- `releases/README.md` 新增发布规范（SemVer、release note 模板、tag 指南）。

## Changed

- `game-concept-architect` 从 “idea→seed→brief” 升级为以下闭环：

```text
idea / sample
-> player verbs
-> action-goal alignment
-> uncertainty calibration
-> system dynamics
-> content flow
-> audience desire
-> playable theme
-> scope/feasibility validation
```

- `game-experience-analyzer` 的输出从单一“体验问题表”升级为“时间线-证据-机制边界-验收动作”四层结构。
- `v0.2.0` 后续版本统一采用 `Added / Changed / Fixed / Safety / Validation` 的发布分段。

## Fixed

- 纠正了 release note 结构碎片化的问题，让 `v0.3.0` 与仓库其它 release 文档风格一致。
- 强化了评估提示中对“竞品可迁移性”和“证据边界”的限定，减少生成偏差。

## Validation

Run from the repository root:

```text
python scripts/validate_repo.py
python scripts/validate_skill.py game-concept-architect
python scripts/validate_skill.py game-experience-analyzer
```

## Safety

- 未引入源书正文、页面图片、私有案例或未授权资料。
- 公开文档与输出行为继续遵循 `CONTRIBUTING.md` 的边界规则。

- 建议补齐历史 tag：`v0.2.0` 挂在 `201b24d`（当时合并 `v0.2` 发布素材）。
