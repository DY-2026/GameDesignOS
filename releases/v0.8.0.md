# v0.8.0 - Runtime Foundation and VOI Decision Gate

## Release Theme

From portable skill packages to a decision-first project workspace.

GameDesignOS v0.8.0 adds the first OS-level runtime structure around the existing seven skills and upgrades `paranoia-ai-system-evolver` with a complete Value of Information decision gate. The system now asks not only which skill should run, but also which information action is worth paying for and when research must stop.

## Highlights

- Adds `runtime/workspace-template/` for real or synthetic game projects.
- Formalizes `Skill Kernel + Contract Layer + Project Workspace + Runtime Interface`.
- Adds workspace contracts for project manifests, design-asset indexes, decision logs, and information-value assessments.
- Documents five project flows: decision-to-information, idea-to-validation, media-to-diagnosis, weekly ED experiment, and evidence-to-proposal.
- Upgrades adapter guidance from skill loading to workspace-aware, decision-first routing and asset writeback.
- Defines the future local CLI surface, including a planned `gamedesignos voi` command, without claiming that a full CLI ships in this release.
- Extends repository validation to protect the runtime foundation and VOI behavior surface.

### VOI Decision Gate

`paranoia-ai-system-evolver` now requires these fields before broad research, retrieval, memory reads, experiments, or additional AI branches:

```text
decision question
real options
current default action
decision deadline
stakes and reversibility
boundary status: undefined | far | near | locked
target uncertainty
possible signals -> posterior update -> action if seen
EVPI ceiling and practical EVSI
total information cost
smallest positive-net-VOI probe
stop rule
```

If every plausible signal leads to the same action, current decision VOI is treated as near zero. The workflow should act, timebox model learning, or classify the activity as information consumption rather than continue open-ended research.

### Compatibility

Existing skill folders remain independently installable and callable. The workspace and VOI layers are additive. Existing projects do not need to convert all historical files; apply the decision-first contract to new or high-impact decisions first.

### Non-Goals

This release does not ship a hosted API, credential manager, GUI dashboard, or complete CLI binary. It does not add an eighth public skill and does not place real private projects in the public repository. Approximate high/medium/low VOI scoring is an engineering heuristic, not a substitute for a full statistical decision model in high-stakes domains.

### Safety

- Real production work stays in a private workspace or private overlay.
- Public fixtures remain synthetic, public, cleared, or explicitly `needs_review`.
- Human Gates remain required for project commitments, publication, money, production changes, and irreversible actions.
- The runtime does not hold API keys or silently upload project files.
- Local negative evidence, failure details, and unsupported-claim boundaries must not be smoothed into generic AI prose.
