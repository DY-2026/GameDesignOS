# v0.5.0 - Contracted Workflow Governance

This is a Markdown release draft only. Do not create the GitHub release or Git tag until the final release commit is reviewed and validated.

## Highlights

- Repositioned `ParanoiaSkills` as a public base library for AI-assisted game design production and workflow governance.
- Made cross-skill contracts part of the public story: concept promises, validation plans, evidence indexes, issue cards, ED handoffs, and router rules.
- Brought WOOP into the outward-facing workflow-evolution language as task admission, outcome evaluation, failure-pattern detection, and if-then recovery.
- Rebuilt the GitHub landing banner with the project name, one-line positioning, and `v0.5.0` marker in the first viewport.

### Added

- New overview banner: `assets/paranoia-skills-overview-banner-v5.png`, with editable HyperFrames source in `assets/hyperframes/paranoia-banner-v5/`.
- Version badge in the README trio.
- Release note for `v0.5.0`.

### Changed

- Updated `README.md`, `README.zh-CN.md`, and `README.en.md` so the opening explains the current six-skill architecture plus the contract layer.
- Updated GitHub About copy to: `Portable agent skills for game design: evidence diagnosis, concept contracts, ED experiments, and eval-backed AI workflow evolution.`
- Updated the public narrative from a skill list toward an interoperable workflow system:
  - media evidence -> evidence index -> issue cards -> ED handoff -> weekly experiments
  - one-line idea -> player-promise contract -> validation plan -> later media diagnosis
  - workflow change -> WOOP Task Card -> VOI/OODA probe -> eval -> Human Gate -> rollback

### Safety

- Public examples remain synthetic, public, cleared, or marked `needs_review`.
- Real projects, client work, private screenshots, private recordings, local paths, and private overlays remain outside the public repository.
- The new banner is a project-owned generated/edited visual asset and does not depend on external product screenshots or private material.
