# v0.9.0 — Local Runtime Prototype

## Highlights

- Ships the first executable `gamedesignos` CLI.
- Turns the v0.8 workspace and contract foundation into deterministic local commands.
- Keeps model calls, credentials, uploads, and project commitments outside the runtime.
- Supports existing v0.8 workspace schemas without destructive migration.

### Added

- Added an installable Python package with `gamedesignos` and `python -m gamedesignos` entrypoints.
- Added `init`, `status`, `voi`, `route`, `new`, `validate`, `pack`, and `doctor` commands.
- Added conservative workspace generation with lifecycle directories, an empty asset index, decision log, and Decision Object template.
- Added explainable route recommendations with missing-upstream behavior for proposal and ED workflows.
- Added qualitative VOI creation/review that enforces a Decision Object, signal-to-action mapping, at most three information actions, and a stop rule.
- Added draft asset creation and index registration for ten initial asset types.
- Added structural and optional JSON Schema validation.
- Added allowlisted review packs with source-status filtering and SHA-256 records.
- Added automated runtime tests across private and public-synthetic workspaces.
- Added a new generated runtime hero image for the public README: `assets/gamedesignos-runtime-v09-hero.png`.

### Changed

- Decoupled workspace schema version `0.8.0` from runtime implementation version `0.9.0`.
- Updated the project-workspace schema to accept v0.8 and v0.9 runtime declarations.
- Updated runtime documentation from planned command contracts to actual executable behavior.
- Updated the README trio to present v0.9.0 as an executable local runtime rather than only a Runtime Foundation.
- Preserved direct skill installation and v0.8 workspace compatibility.

### Safety

- `--force` never overwrites an existing GameDesignOS workspace.
- Draft assets contain TODO markers and are never marked accepted.
- `route` recommends a skill but does not execute one.
- `voi` makes no automatic numeric expected-value claim.
- Public packs include only registered assets allowed by the selected source-status mode; raw decision logs, owners, and unfiltered indexes are excluded.
- The runtime stores no API keys and performs no model or network call.
