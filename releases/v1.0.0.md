# GameDesignOS v1.0.0 — Formal Project-Ready Release

> Release status: formal 1.0 baseline, ready for real-project validation, public onboarding, and maintainer review.

v1.0.0 不再把 GameDesignOS 定位为“能初始化和打包的本地 prototype”，而是推进到正式 Project-Ready：一个真实游戏项目可以用它连续管理从想法、假设、证据、实验、决策到复盘的完整周期。

## 对外定位

GameDesignOS v1.0.0 的公开一句话是：

> 把一次 AI agent 会话，变成可持续推进真实项目的游戏设计操作系统。

英文核心描述：

> GameDesignOS is a local-first operating system for AI-assisted game design. It turns AI-agent sessions into structured decisions, assumptions, evidence, experiments, proposals, workflows, and learning records, so teams can validate ideas, diagnose gameplay, and evolve production workflows without losing context or bypassing human judgment.

本版本对外强调 7 个专家 skill、17 个 contract schema、5 条端到端工作流、v1 项目 workspace 和 1 个确定性本地 runtime。它不是自动替人做策划的 autopilot，而是让 Codex、Claude Code、OpenCode 等 agent 在同一套证据、契约、Human Gate 和 rollback 边界内协作。

## 核心变化

- 默认 runtime 版本升级到 `1.0.0`。
- 新建 workspace 默认使用 schema `1.0.0`。
- 保留 v0.8/v0.9 workspace 兼容，不强制迁移旧项目。
- GitHub 首页、About metadata、PyPI 包描述与产品愿景统一到正式 1.0 口径。
- 新增正式首页图 `assets/gamedesignos-v1-hero.svg`：包含项目名、英文核心定位、核心资产数字和本地优先/Human Gate 安全边界；可见文字全部使用英文 SVG 矢量文本，避免 README 缩放后发虚或跨平台字体不一致。

## 行为门

- 无 Decision Object 时，VOI Gate 阻止 research。
- 高影响 decision 缺 rollback 时，Commitment/Rollback Gate 阻断。
- 高风险 assumption 未测试时，Decision 不可 accept。
- Experiment result 未 review 时，Decision 不可 accept。
- Decision accept 必须显式提供 `--by` 与 `--reason`。
- public-synthetic pack 不导出 private evidence。

## 对标吸收

- 首屏用一句话说明“为什么存在”，再用可数资产说明“仓库里到底有什么”。
- 把 Project-Ready 写成正式 release baseline，而不是候选口径。
- 将用户控制、Human Gate、本地优先和公开/私有分离放在介绍前半段，避免被误读为自动化替代策划。
- 保留 GameDesignOS 自身的证据、契约、决策和 workflow-governance 差异化，不把项目包装成通用 multi-agent studio 模板。

## Rollback

如果 v1.0 需要回退：

1. 保留 v0.8/v0.9 workspace，不做迁移。
2. 删除或停用 `gamedesignos/project_ready.py`。
3. 从 `gamedesignos/cli.py` 移除 v1 分组命令。
4. 将 `pyproject.toml` 与 `RUNTIME_VERSION` 回退到 `0.9.0`。
5. 保留新增 v1 schema 作为候选文档，直到 maintainer 决定删除或继续迭代。

不得通过移动项目根目录、替换 `.git` 或镜像同步来 rollback。
