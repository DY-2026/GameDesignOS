# GameDesignOS v1.0.0 — Project-Ready GameDesignOS

> Release status: local runtime complete, ready for maintainer review and real-project validation.

v1.0.0 不再把 GameDesignOS 定位为“能初始化和打包的本地 prototype”，而是推进到 Project-Ready：一个真实游戏项目可以用它连续管理从想法、假设、证据、实验、决策到复盘的完整周期。

## 核心变化

- 默认 runtime 版本升级到 `1.0.0`。
- 新建 workspace 默认使用 schema `1.0.0`。
- 保留 v0.8/v0.9 workspace 兼容，不强制迁移旧项目。
- 新增 v1 workspace 结构：

```text
00-inbox/
01-decisions/
02-assumptions/
03-evidence/
04-experiments/
05-design-assets/
06-workflows/
07-learning/
08-exports/
.gamedesignos/
```

## 新增 CLI 主路径

```text
gamedesignos ask
gamedesignos "<one sentence>"
gamedesignos start
gamedesignos decision new/list/inspect/accept/reject/supersede
gamedesignos assumption new/list/validate
gamedesignos evidence add/list/inspect
gamedesignos experiment plan/result/review
gamedesignos gate run
gamedesignos workflow list/start/status/next/validate
gamedesignos health
gamedesignos next
gamedesignos graph export/inspect
```

## 行为门

- 无 Decision Object 时，VOI Gate 阻止 research。
- 高影响 decision 缺 rollback 时，Commitment/Rollback Gate 阻断。
- 高风险 assumption 未测试时，Decision 不可 accept。
- Experiment result 未 review 时，Decision 不可 accept。
- Decision accept 必须显式提供 `--by` 与 `--reason`。
- public-synthetic pack 不导出 private evidence。

## Rollback

如果 v1.0 需要回退：

1. 保留 v0.8/v0.9 workspace，不做迁移。
2. 删除或停用 `gamedesignos/project_ready.py`。
3. 从 `gamedesignos/cli.py` 移除 v1 分组命令。
4. 将 `pyproject.toml` 与 `RUNTIME_VERSION` 回退到 `0.9.0`。
5. 保留新增 v1 schema 作为候选文档，直到 maintainer 决定删除或继续迭代。

不得通过移动项目根目录、替换 `.git` 或镜像同步来 rollback。
