# EPM 体验密度框架

EPM，Experience per Minute，指玩家在单位时间内获得的有效体验信号密度。这里的体验不是单纯事件数量，而是玩家能理解、能回应、能形成状态变化或情绪波峰的设计信号。

一个有效体验信号通常同时满足四个条件：玩家看得见，反馈能理解，状态有变化，接下来短时间内会影响目标、选择、节奏或情绪。只有飘字、红点、弹窗、弱奖励、重复对白，而没有行为变化，不计为高质量 EPM。

## 三个主旋钮

### 1. Event Frequency

事件频率关注玩家多久遇到一次可感知变化。

可计入的事件包括：

- 战斗波峰、敌人组合变化、Boss 预告、危险提示。
- 掉落、升级、临时增益、Build 组件、稀有奖励。
- 新目标、短链任务完成、地图/环境变化。
- 教学完成、功能暴露、功能解锁、首次使用。
- 微剧情、角色反应、局内叙事反馈。

常见问题是长时间没有有效状态变化，或者事件很多但强度全一样。一个实用节奏是弱事件保持连续感，强事件承担记忆点，避免全程平铺。

### 2. Decision Weight

决策权重关注一次选择是否改变接下来 30-120 秒的体验。

有效决策不等于选项多。弱决策通常是三个奖励里随便点一个，玩家看不出差异，也不改变下一段玩法。强决策会改变敌人、路线、资源、Build、风险、叙事态度或任务目标，并且让玩家在短时间内看到结果。

判断一个决策是否有重量，可以问四个问题：

| 问题 | 说明 |
| --- | --- |
| 是否改变接下来可见体验 | 不是只改变远期数值 |
| 玩家是否能理解差异 | 选项文案和反馈是否清楚 |
| 结果是否能被归因 | 玩家知道这是自己选择导致的 |
| 是否保留再选择空间 | 不是一次选择后完全锁死 |

### 3. Narrative Compression

叙事压缩关注信息、情绪和设定是否以更少停顿交付。

压缩不是删剧情，而是删重复解释、弱停顿和不可交互等待。优先用镜头、动画、环境变化、角色短反应、任务文本、战斗中浮字和结算卡传递信息。玩家仍然要获得情绪，但不应该在早期循环建立前被长文本截断。

## EPM 问题分类

| 类型 | 症状 | 常见原因 | 优先旋钮 |
| --- | --- | --- | --- |
| low_density | 30-60 秒没有状态变化或目标推进 | 等待、跑图、教程空窗、奖励过稀 | event_frequency |
| false_density | 事件很多但不影响行为 | 弱飘字、重复奖励、红点堆叠、无差异选项 | decision_weight |
| overload_density | 玩家来不及理解或操作 | 教学、掉落、剧情、UI 同时砸过来 | narrative_compression |
| unreadable_density | 玩家看到变化但不知道意义 | 反馈弱、归因差、目标不清、文案不直观 | feedback_intensity / objective_chain |

## 计算口径

EPM 不是单一真理指标，建议用于对比同一版本、同一用户阶段、同一玩法段落。

```text
EPM = meaningful_event_count * average_feedback_clarity * average_player_relevance / playable_minutes
```

其中：

- `meaningful_event_count`：有效体验信号数量。
- `average_feedback_clarity`：玩家是否能理解发生了什么，0-1。
- `average_player_relevance`：事件是否影响目标、选择、成长或情绪，0-1。
- `playable_minutes`：可玩时间，不含加载、崩溃、不可跳过长等待。

这个公式只用于设计讨论和相对比较，不要包装成严谨统计模型。真实实验仍然要看 D1/D3/D7、会话时长、退出点、失败率、负反馈和转化路径。

## 诊断步骤

1. 先画 5-10 分钟体验时间轴。
2. 标注每个有效事件、弱事件、强事件和空窗。
3. 标注玩家决策点，并写清影响窗口。
4. 标注叙事停顿和不可交互段落。
5. 找出最长无意义空窗、最高过载点和最弱决策段。
6. 只选一个主旋钮进入本周实验。

## 输出建议

每个诊断结论都应落到可修改对象：关卡刷怪表、奖励表、教程触发、叙事包、UI 提示、Build 选项、任务链、埋点事件或看板字段。不要只写上头感不足、反馈不够爽、节奏偏慢这类无法交接的句子。
