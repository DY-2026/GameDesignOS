# 一周 ED 实验 SOP

一周实验的目标不是证明一个大版本方向完全正确，而是在低成本、低风险、可回滚的范围里，判断一个体验浓度旋钮是否值得放大。

## 周期节奏

| 时间 | 目标 | 产物 |
| --- | --- | --- |
| 周一 | 冻结假设和变体 | 最佳刺激窗口假设、实验假设、A/B/C/D 变体、指标口径、Kill 条件 |
| 周二 | 上测试服或灰度 | 开关位、埋点、QA 通过、分流规则 |
| 周三 | 检查阻断项 | 崩溃、埋点缺失、样本异常、明显体验事故 |
| 周四 | 小复查 | TTE、ED proxy、退出点、负向反馈早期趋势 |
| 周五至周日 | 收样本 | 分群数据、会话数据、选择后窗口数据 |
| 下周一 | 决策 | 放大、继续观察、回滚、二次实验 |

## 实验设计原则

每个变体只保留一个主旋钮。一次同时降噪、改反馈、调操控、改氛围、加选择，很容易让结果不可归因。变体必须先写 `optimal_stimulation_target`，说明它服务刺激不足、过载、习惯化、低能动性还是低意义感。默认结构如下：

| 组别 | 目的 | 最佳刺激目标 | 主旋钮 |
| --- | --- | --- | --- |
| A | 对照组 | current | 不改 |
| B | 测试降低认知负荷是否改善体验 | reduce_overload / add_familiar_anchor | CLP |
| C | 测试提升瞬时反馈质量是否改善体验 | improve_attributable_interest | SF / EB / AR |
| D | 测试调整有意义选择频率是否改善体验 | add_semi_novelty_only_if_window_allows | MD/min |

如果团队资源很小，先跑 A/B。B 组选择最可能影响目标指标的主旋钮，不要为了形式强行做三组。

如果问题是老玩家、赛季、刷子、肉鸽、UGC 或重复日常疲劳，可以增加可选 E 组做反习惯化，但仍然只保留一个主旋钮，并输出 `anti_habituation_plan`。

## 样本与分流

优先分新用户、回流用户、老用户。新用户看首局激活和 D1，回流用户看进入后目标恢复和第二会话，老用户看疲劳段和版本扰动。不同渠道、平台、版本、付费状态和账号年龄不要混在一起做结论。

最小样本量取决于产品体量。这个 skill 不强行给统计显著性承诺，但要求写清 `minimum_sample_note`：样本不足时只允许判断方向性风险，不能做全量放大。

## 每日检查

周三必须先看质量门，不要直接庆祝指标变化。

| 检查项 | 风险 |
| --- | --- |
| crash_rate / anr_rate | 技术问题会污染留存 |
| assignment_balance | 分流不均会污染判断 |
| event_missing_rate | 埋点缺失会让 ED proxy 失真 |
| version_mismatch | 客户端版本混杂会污染判断 |
| early_exit_spike | 变体可能制造强反感 |
| difficulty_spike | 事件变多可能实际变难 |
| optimal_stimulation_mismatch | “无聊”可能是过载、习惯化、低能动性或低意义感，而不是刺激不足 |
| novelty_overload | 新颖性可能变成随机、噪声或不可归因规则 |

## 决策分层

复盘时先看负向门，再看 P1 指标，最后看解释性指标。负向门触发时，即使 D1 上升也不要直接放大。

| 决策 | 条件 |
| --- | --- |
| amplify | P1 达标，负向门未触发，最佳刺激窗口与解释性指标支持设计假设 |
| iterate | P1 未达标但 P2 明显改善，且负向风险可控 |
| observe | 样本不足或指标方向不一致 |
| rollback | 负向门触发，或埋点/技术质量无法支持判断 |
| kill | 改动方向与玩家行为证据相反，继续优化成本不划算 |

## 交接清单

策划负责假设、变体和成功规则。客户端负责开关位、埋点和配置读取。关卡或叙事负责可上线内容。数据负责看板和分群。QA 只需要验证可玩、分流、埋点触发和回滚，不应承担完整大版本回归。

## 常见失败

最常见失败是为了让实验看起来完整，强行做多个大改。第二类失败是埋点只记曝光，不记玩家选择后窗口。第三类失败是只看平均会话时长，忽略退出点和负反馈。第四类失败是 D 组叙事压缩做成剧情删除，导致情绪承诺丢失。
