# 交互、预测与反馈镜头

这个镜头用于解释为什么游戏相比短视频仍然有独特优势：游戏不是只把高密度内容推给玩家，而是让玩家通过行动参与预测、反馈和修正。短视频的浓缩主要来自剪辑、信息和情绪峰值；游戏的浓缩则可以来自交互闭环。

本文档使用预测处理、自由能、马尔可夫毯、具身感和受控幻觉作为设计隐喻，而不是科学证明。它的作用是帮助设计师提出更好的反馈问题，不能替代用户研究、心理学实证或数据验证。完整的 FEP/马尔可夫毯字段见 `references/free-energy-markov-blanket-lens.zh-CN.md`。

## 交互的优势

视频可以高频提供新信息，但玩家只能接收。游戏可以让玩家先形成意图，再执行动作，再看到世界回应。这个回路会让反馈不只是刺激，而是和自我行动绑定在一起。

因此，体验浓度不应该只追求每分钟更多事件，而要追求处在最佳刺激窗口内的可归因交互反馈。玩家需要感觉到：我预测了，我行动了，世界回应了，我修正了下一次行动。

从 FEP 镜头看，这个循环在管理预测误差：给玩家一点可控惊讶，再给足解释和修正空间。从马尔可夫毯镜头看，这个循环在检查体验边界：玩家通过行动状态影响世界，通过感官状态理解世界变化。

如果玩家说“无聊”，先不要直接加事件。先判断这是不是 `under_stimulation`、`over_stimulation`、`habituation`、`low_agency` 或 `low_meaning`。交互闭环的价值，是让刺激能被玩家吸收和使用；如果玩家不能行动、不能归因、不能复盘，再多事件也只是噪声。

## 预测循环

一个高质量交互反馈可以拆成四段：

| 阶段 | 设计问题 | 例子 |
| --- | --- | --- |
| 预测 | 玩家是否知道自己想达成什么 | 闪避危险、打断 Boss、选择构筑方向 |
| 行动 | 输入是否稳定且符合意图 | 按键响应、触屏拖拽、方向控制 |
| 反馈 | 世界是否清楚回应行动 | 命中停顿、音效、特效、敌人硬直、资源变化 |
| 修正 | 玩家是否能据此调整下一步 | 改变站位、换技能、升级路线、选择风险 |

如果反馈不能被归因，玩家就很难形成下一次预测。此时增加事件频率只会制造噪音。

## 边界耦合

诊断手感和反馈时，不要只写“更爽一点”。要明确是哪一层边界断了：

| 边界层 | 设计问题 | 例子 |
| --- | --- | --- |
| 感官状态 | 世界变化是否准确返回 | UI 状态、敌人硬直、音效、震动、镜头、地图提示 |
| 行动状态 | 玩家意图是否能稳定表达 | 按键、摇杆、拖拽、Build 选择、资源分配、路径选择 |
| 耦合关系 | 行动和反馈是否低延迟、低噪声、高解释性 | 命中后立即有同频反馈，失败后能知道下一次怎么改 |

## 真实幻觉与具身感

游戏中的真实感不是等同现实，而是让玩家的大脑暂时接受一套稳定的输入-反馈规律。具身感来自控制、镜头、角色和世界规则之间的连续一致。

提升具身感可以从这些地方入手：输入响应、动画起手和收招、命中停顿、镜头惯性、物理反馈、震动、角色状态可读性。目标不是写更多解释，而是让玩家身体先相信自己做到了。

## 注意力调度

持续体验不是一直刺激，而是不断让玩家的注意力有可以落脚的对象。

| 注意力对象 | 设计手段 |
| --- | --- |
| 目标 | 下一步目标、短链任务、危险预告 |
| 身体 | 操控节奏、镜头、命中反馈、移动手感 |
| 结果 | 掉落、状态变化、敌人反应、剧情回应 |
| 意义 | 角色态度、世界变化、选择后果、主题表达 |

好的体验浓度会在这些对象之间切换，而不是让所有通道同时轰炸玩家。

## 设计检查

输出诊断时，至少回答四个问题：

1. 玩家此刻能预测什么？
2. 玩家此刻能行动改变什么？
3. 玩家是否清楚看到世界回应？
4. 玩家是否能根据反馈修正下一步？
5. 当前预测误差是太低、适中，还是太高？
6. 玩家-游戏边界的断点是延迟、噪声、映射不清，还是能动性弱？
7. 当前刺激窗口是太低、适中、过载，还是分群不均？

如果这些问题中任何一环断裂，体验浓度就会被稀释。修复顺序通常是先判最佳刺激窗口，再降认知负荷，再提高反馈归因，再增加选择频率。
