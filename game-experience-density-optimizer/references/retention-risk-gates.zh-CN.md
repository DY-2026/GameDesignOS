# 指标实验风险门

体验浓度实验很容易被短期指标诱导。这个文件用于在实验前写死成功、观察、回滚和 Kill 条件，避免事后挑指标解释。单机/买断制和手游/liveops 必须使用不同的 P1 周期。

## 预注册规则

实验上线前必须写清：

- `game_metric_model`：`premium_single_player`、`mobile_liveops`、`hybrid` 或 `unknown`。
- `primary_success_metric`：最重要的结果指标。单机/买断制可以是总有效游玩时长、章节/Demo 完成率、核心循环到达率；手游/liveops 可以是 D1/D7、每日会话或活动留存。
- `secondary_metrics`：解释性指标，例如 ED proxy、TTE、CLP、SF、EB、AR、MD/min、决策后窗口行为。
- `optimal_stimulation_fit`：预注册当前假设是刺激不足、过载、习惯化、低能动性、低意义感，还是未知；并写明错误时的风险。
- `negative_metrics`：任何触发后必须暂停或回滚的指标。
- `minimum_sample_note`：样本不足时如何解释。
- `decision_date`：哪一天复盘，不无限延期。
- `rollback_owner`：谁有权限回滚。

## 默认成功门

以下阈值是起手建议，不是强制统计承诺。真实项目可以按体量改，但必须在实验前固定。

| 模型 | 成功 | 可迭代 | 观察 | 回滚 / Kill |
| --- | --- | --- | --- | --- |
| `premium_single_player` | 总有效游玩时长、Demo/章节完成率、核心循环到达率或重玩意愿达到预注册改善，负向门未触发，且最佳刺激窗口假设未被证伪 | ED proxy、TTE、CLP、SF/EB/AR、MD/min、optimal_stimulation_band 中至少两项支持假设，P1 尚未显著改善但负向风险可控 | 样本不足、章节样本不完整、P2 与总旅程 P1 不一致 | 退出点提前、失败率/投诉/退款或负评风险上升、不可跳过流程变长、过载无聊上升 |
| `mobile_liveops` | D1/D7、每日会话、连续活跃、活动留存或回流成功率达到预注册改善，负向门未触发，且疲劳/习惯化信号未恶化 | ED proxy、TTE、CLP、SF/EB/AR、MD/min、optimal_stimulation_band、boredom_type_mix、anti_habituation_fit 中至少两项支持假设，P1 尚未显著改善但负向风险可控 | 样本不足、D1/D7 方向不稳定、P2 与 P1 不一致 | 早退、崩溃、失败率、投诉、付费误触、经济异常、日常疲劳、习惯化或过载无聊触发负向门 |
| `hybrid` | 总旅程 P1 和 liveops P1 至少不互相伤害，目标主指标改善 | 一套周期改善但另一套未受损，可继续独立实验 | 两套周期冲突或样本不足 | 任一关键周期明显受损，或暗黑模式风险 |

## 负向门

| 风险 | 触发信号 | 动作 |
| --- | --- | --- |
| 技术风险 | crash/anr 明显上升 | 立即暂停判断，必要时回滚 |
| 难度风险 | 失败率、死亡率、卡关率明显上升 | 回滚或降级变体 |
| 经济风险 | 奖励产出、货币消耗、付费转化异常 | 暂停放大，检查经济表 |
| 叙事风险 | 跳过率高且跳过后留存下降 | 重新压缩，不继续删剧情 |
| 新手风险 | 新用户早退上升 | 新手分流回滚 |
| 老玩家风险 | 老玩家投诉或核心循环扰动 | 老玩家排除或独立实验 |
| 公平风险 | PVP、排行榜、付费竞争受影响 | 禁止灰度或只在非竞争环境测试 |
| 暗黑模式风险 | 利用误导、焦虑、虚假稀缺 | Kill，不进入实验 |
| 最佳刺激误判风险 | 玩家无聊反馈中 `over_stimulation`、`habituation`、`low_agency` 上升 | 暂停放大，回到 OLSO 诊断，不继续加事件 |
| 新颖性过载风险 | `optimal_novelty_fit=too_high`、困惑/投诉/跳出上升 | 降低随机和信息层，增加熟悉锚点 |

## 分群要求

至少区分 `new_user`、`returning_user`、`existing_user`。如果样本量允许，进一步按平台、渠道、版本、付费状态、首局/非首局分层。不要用混合均值掩盖某个关键群体受损。

## 不可接受的胜利

这些情况不能算成功：

- 会话时长上升，但退出点更靠后只是因为不可跳过流程变长。
- D1 上升，但失败率、投诉或差评明显上升。
- 单机/买断制里把 D1/D7 当成默认胜利，而不看总游戏时长、章节推进或完成率。
- 手游/liveops 里只看总时长上升，而不看持续天数、日常疲劳和回流质量。
- ED proxy 上升，但 meaningful decisions 或 salient feedback 实际只是弱弹窗和红点。
- 选择次数上升，但选择没有可见后果。
- “无聊”下降来自更高压力或更强义务，而 `over_stimulation`、投诉、疲劳或退出也上升。
- 新颖性上升，但只是随机、换皮、信息轰炸或不可归因规则跳变。
- 老玩家短期回流，但 `habituation` 没被修复，只是靠奖励补贴延后疲劳。
- 跳过率下降，只是因为取消了跳过按钮。
- 付费转化上升，来自误导性 UI 或焦虑式限时。

## 回滚策略

每个变体都应通过配置开关关闭。不能要求重新发大包才能回滚。回滚后仍要保留分流和埋点记录，标注 `rollback_reason`，避免后续复盘找不到原因。
