# 留存实验风险门

留存实验很容易被短期指标诱导。这个文件用于在实验前写死成功、观察、回滚和 Kill 条件，避免事后挑指标解释。

## 预注册规则

实验上线前必须写清：

- `primary_success_metric`：最重要的结果指标，例如 D1、首局关键 checkpoint 到达率。
- `secondary_metrics`：解释性指标，例如 EPM、TTE、决策后窗口行为。
- `negative_metrics`：任何触发后必须暂停或回滚的指标。
- `minimum_sample_note`：样本不足时如何解释。
- `decision_date`：哪一天复盘，不无限延期。
- `rollback_owner`：谁有权限回滚。

## 默认成功门

以下阈值是起手建议，不是强制统计承诺。真实项目可以按体量改，但必须在实验前固定。

| 判断 | 默认条件 |
| --- | --- |
| 成功 | D1 提升至少 1.0 个百分点，且会话中位时长提升至少 8%，负向门未触发 |
| 可迭代 | EPM 提升至少 15%，TTE 降到 20 秒以内，P1 尚未显著改善但负向风险可控 |
| 观察 | 样本不足、D1 方向不稳定、P2 与 P1 不一致 |
| 回滚 | 早退、崩溃、失败率、投诉、跳过后流失或经济异常触发负向门 |
| Kill | 改动提高指标但违背产品承诺、暗黑模式风险高、或需要大范围重构才可能成立 |

## 负向门

| 风险 | 触发信号 | 动作 |
| --- | --- | --- |
| 技术风险 | crash/anr 明显上升 | 立即暂停判断，必要时回滚 |
| 难度风险 | 失败率、死亡率、卡关率明显上升 | 回滚或降级变体 |
| 经济风险 | 奖励产出、货币消耗、付费转化异常 | 暂停放大，检查经济表 |
| 叙事风险 | 跳过率高且跳过后留存下降 | 重新压缩，不继续删剧情 |
| 新手风险 | 新用户早退上升 | 新手分流回滚 |
| 老玩家风险 | 老玩家投诉或核心循环扰动 | 老玩家排除或独立实验 |
| 公平风险 | PVP、排行榜、付费竞争受影响 | 禁止灰度或只在非竞争环境测试 |
| 暗黑模式风险 | 利用误导、焦虑、虚假稀缺 | Kill，不进入实验 |

## 分群要求

至少区分 `new_user`、`returning_user`、`existing_user`。如果样本量允许，进一步按平台、渠道、版本、付费状态、首局/非首局分层。不要用混合均值掩盖某个关键群体受损。

## 不可接受的胜利

这些情况不能算成功：

- 会话时长上升，但退出点更靠后只是因为不可跳过流程变长。
- D1 上升，但失败率、投诉或差评明显上升。
- EPM 上升，但 meaningful events 实际只是弱弹窗和红点。
- 选择次数上升，但选择没有可见后果。
- 跳过率下降，只是因为取消了跳过按钮。
- 付费转化上升，来自误导性 UI 或焦虑式限时。

## 回滚策略

每个变体都应通过配置开关关闭。不能要求重新发大包才能回滚。回滚后仍要保留分流和埋点记录，标注 `rollback_reason`，避免后续复盘找不到原因。
