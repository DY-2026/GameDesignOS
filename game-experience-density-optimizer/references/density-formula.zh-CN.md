# 体验浓度公式

本文档把体验浓度从一句定性描述，压缩成一个可用于诊断、讨论和原型调优的工作公式。它不是经过科学验证的心理学量表，也不应被包装成严谨统计模型。它的价值在于帮助设计师把体验问题拆成可观察、可讨论、可修改的设计变量。

## 基础定义

体验浓度，Experience Density，简称 ED，指单位时间内玩家所经历的有意义选择与可感知反馈共同构成的游戏体验浓缩程度。

更口语化地说，它关注每分钟有多少事情真正发生，以及这些事情是否被玩家感知、理解、归因并转化为继续行动的动力。

这个概念的背景不是“游戏必须学短视频”，而是移动互联网和短视频把快节奏、高浓缩推成了内容默认值后，游戏需要重新说明自己的优势：交互。视频主要通过剪辑和信息压缩调动注意力，游戏可以通过预测、行动、反馈、修正的闭环，让玩家参与一个更强的受控幻觉。

因此，ED 公式只能作为设计启发式使用。它可以帮助讨论和落地，但没有科学验证，不能包装成严谨心理学或统计模型。

## OLSO 解释：最佳刺激窗口中的体验浓度

在使用 ED 公式前，先判断玩家是否处在合适的刺激窗口。最佳刺激水平模型（Optimal Level of Stimulation / OLSO）在这里不是量表，而是一个设计镜头：不同玩家、阶段、场景和平台，会有不同的可承受刺激范围。

升级后的定义是：

```text
体验浓度 = 当前玩家在当前情境下，单位时间内可吸收、可解释、可转化为探索/学习/意义的刺激密度
```

这意味着“无聊”不一定等于刺激太少。玩家说无聊时，至少要先区分：

| 类型 | 典型表现 | 公式含义 | 设计方向 |
| --- | --- | --- | --- |
| `under_stimulation` | 淡、空、重复、没有新东西 | `MD/min` 太低，或 SF/AR 没有新信息 | 增加半熟半新的可控惊讶 |
| `over_stimulation` | 太乱、太累、看不懂，所以不想碰 | `CLP` 太高，刺激超过可吸收窗口 | 降噪、加熟悉锚点、拆步 |
| `habituation` | 老玩家已经完全预测流程 | 新信息和旧模型的差异消失 | 换用法、换语境、换组合深度 |
| `low_agency` | 事情很多，但玩家影响不了结果 | `MD/min` 是假选择，行动状态弱 | 让选择改变状态、风险或后续路径 |
| `low_meaning` | 奖励很多，但不值得关心 | SF/AR 与目标、叙事或身份无关 | 把反馈接回目标、世界和角色意义 |

因此，公式只回答“主旋钮在哪里”，不能直接回答“刺激应该更多还是更少”。方向必须先由 `optimal_stimulation_fit` 决定。

## FEP 解释：可承受预测误差密度

从自由能原理的设计隐喻看，体验浓度可以被压缩成一句话：

```text
体验浓度 = 单位时间内玩家可承受预测误差的密度
```

这个解释不替代 ED 公式，而是帮助理解“为什么不是越快越好”。玩家需要惊讶来更新模型，也需要结构来解释惊讶。预测误差太低，玩家无聊；预测误差太高，玩家崩溃；预测误差适中、可解释、可行动，并且处在该玩家的最佳刺激窗口内，玩家更容易进入心流。

进一步说，游戏设计是在搭一条自由能斜坡：前期帮助玩家建立基本模型，中期制造可理解的新误差，后期让玩家能处理系统层、策略层甚至元博弈层的惊讶。成长不是把数字变大，而是让玩家能承受、解释和利用更复杂的预测误差。

因此，公式中的每一项都要服务“可控惊讶”：

| 项 | 如何管理预测误差 |
| --- | --- |
| `MD/min` | 控制预测误差出现的频率 |
| `SF` | 让误差信号显著、可归因 |
| `EB` | 让行动模型稳定，减少“我明明操作了但世界没听懂”的误差 |
| `AR` | 让世界模型连续，留白也有可解释的质感 |
| `CLP` | 降低不可解释误差、噪声和心智成本 |

## 公式使用顺序

每次使用公式前，先输出 `optimal_stimulation_fit`：

```yaml
optimal_stimulation_fit:
  band: too_low | optimal | too_high | uneven | unknown
  boredom_type: under_stimulation | over_stimulation | habituation | low_agency | low_meaning | mixed | unknown
  player_resource_profile: unknown
  stimulus_profile: unknown
  context_stimulation: unknown
  design_direction: add_semi_novelty | add_familiar_anchor | reduce_overload | anti_habituation | repair_agency | add_meaning | preserve_curve | unknown
```

然后再决定 ED 公式的主旋钮：

| `optimal_stimulation_fit.band` | 公式优先项 | 不要先做 |
| --- | --- | --- |
| `too_low` | 在低 `CLP` 前提下增加 `MD/min` 或强化 `SF/AR` 的新信息 | 粗暴加红点、弱弹窗、纯数值奖励 |
| `too_high` | 降低 `CLP`，修复 `SF/EB` 的归因和边界耦合 | 继续加事件、随机性、惩罚和信息层 |
| `optimal` | 保护曲线，只做小幅提质或分群微调 | 因为短期指标诱惑而过度加料 |
| `uneven` | 分玩家/分阶段处理，避免平均值掩盖问题 | 用一个全局节奏覆盖新手和核心玩家 |
| `unknown` | 设计最小验证，先收证据 | 伪造玩家心理或承诺留存提升 |

## 二维坐标系

体验浓度可以先用一个二维坐标理解：横轴是时间，纵轴是瞬时体验丰富度。

横轴对应水平密度，关注有意义选择发生的频率。它回答的是玩家多久需要主动判断一次，多久会遇到一次能改变局势、收益路径或表达结果的选择。

纵轴对应垂直密度，关注某一瞬间反馈的丰富程度。它回答的是在这一刻，画面、音效、震动、动画、剧情、意义感、身体感和氛围是否共同形成了足够清晰而强烈的体验切片。

## 统一公式

```text
ED = MD/min * (SF + EB + AR) / CLP
```

也可以写成：

```text
体验浓度 = 有意义选择频度 * 瞬时反馈质量 / 认知负荷惩罚
```

其中：

| 缩写 | 名称 | 设计含义 |
| --- | --- | --- |
| `ED` | Experience Density | 体验浓度 |
| `MD/min` | Meaningful Decisions per minute | 每分钟有意义选择次数 |
| `SF` | Salient Feedback | 可被玩家感知并归因的反馈 |
| `EB` | Embodiment Bonus | 操控、镜头、角色和物理一致性带来的具身感加成 |
| `AR` | Atmospheric Richness | 音画、美术、环境、风格一致性带来的氛围感加成 |
| `CLP` | Cognitive Load Penalty | 学习成本、信息噪音、UI 混乱、打断感带来的认知负荷惩罚 |

## MD/min：水平密度

MD/min 是公式的节拍器。它衡量玩家在单位时间内遇到多少真正有后果的选择。

有意义选择不是按钮次数，也不是菜单数量。一次选择必须至少改变局势、收益路径、风险、表达、关系、路线、Build 或接下来一段体验，才算进入 MD/min。

常见例子包括：三选一构筑、关键闪避、姿态切换、资源分配、路线选择、对话立场、战斗目标优先级、风险收益取舍。无意义点击、重复确认、自动寻路中的机械操作、只是换皮的奖励选择，不应计入。

## SF：可感知反馈

SF 是玩家能明确感受到的反馈强度和清晰度。它要求反馈能被感官捕捉，并且玩家知道它和自己的操作或选择有关。

暴击特效、命中音效、清屏演出、升级金光、任务完成提示、Boss 掉落、剧情反转、敌人状态改变，都可以成为 SF。关键是同频共振：视觉、听觉、触觉、镜头、动画最好同时传递同一个核心信息。

## EB：具身感加成

EB 来自玩家和角色、载具、光标、镜头之间的人机一体感。它不是单纯代入故事，而是让玩家在数秒内忘记自己在按按钮，感觉意图被身体化地执行。

响应灵敏的操控、符合预期的起手和收招、清晰的命中停顿、稳定且不晕的镜头、流畅有表现力的动画、触觉反馈和物理一致性，都会提高 EB。

## AR：氛围感加成

AR 来自世界和审美的一致性。它让玩家即使在没有高频决策的时刻，也能感觉自己处在一个自洽、有质感、有风格的空间里。

音乐、环境音、光影、粒子、美术风格、UI 语言、场景调度、角色表演、文本气质，都可能贡献 AR。AR 不一定让节奏更快，但会让留白不空。

## CLP：认知负荷惩罚

CLP 是分母。它代表玩家为了理解游戏而付出的额外心智成本。UI 混乱、规则描述模糊、特效光污染、教学过长、弹窗打断、加载割裂、目标不清、术语堆叠，都会提高 CLP。

当 CLP 很高时，增加 MD/min 和 SF 往往不会让体验更浓，而会让体验更烦躁。玩家看不懂，就没法行动；无法归因，就无法形成预测和反馈循环。

## 使用边界

这个公式优先用于同一游戏、同一阶段、同一用户群体之间的相对比较，不适合跨品类机械打分。高浓度不是绝对好，低浓度也不是绝对差。恐怖、叙事、探索、模拟经营、禅意游戏，都可能通过低水平密度和高氛围密度成立。

真正要判断的是设计意图、玩家偏好和浓度曲线是否匹配。一个游戏可以整体高浓度，但需要留白；也可以整体舒缓，但不能让留白变成空洞和无反馈。
