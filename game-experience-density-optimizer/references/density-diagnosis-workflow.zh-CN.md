# 体验浓度诊断流程

体验浓度不是越高越好。诊断时不要直接问如何让游戏更快，而要先判断当前设计意图需要什么样的浓度曲线，以及玩家现在卡在公式的哪一项。

核心顺序是：先判窗口，再降噪，再提质，后调频。

## 第零步：判断最佳刺激窗口

先判断当前体验到底是刺激不足、刺激过载、习惯化，还是玩家能动性/意义感不足。不要把所有“无聊”都自动解释为“要加更多事件”。

| 窗口 | 玩家反馈 | 可能成因 | 下一步 |
| --- | --- | --- | --- |
| `too_low` | 淡、空、没事做、太可预测 | 刺激低于玩家当前资源和期待 | 增加半熟半新的可控惊讶，强化选择后果 |
| `optimal` | 想再试、能解释失败、每局有小变化 | 刺激强度和玩家资源匹配 | 保护曲线，谨慎微调 |
| `too_high` | 太乱、太累、看不懂、不想碰 | 刺激超过可吸收范围，或信息噪声过高 | 先降 CLP，增加熟悉锚点和可归因反馈 |
| `uneven` | 新手崩溃、老手无聊，或某阶段突然断裂 | 玩家段/阶段/平台的刺激窗口不同 | 分群或分段设计，不用平均值覆盖 |
| `unknown` | 证据不足 | 不能判断 | 写明未知项，设计最小验证 |

同时标注 `boredom_type`：

| boredom_type | 识别问题 | 优先动作 |
| --- | --- | --- |
| `under_stimulation` | 玩家能看懂，但没有新预测误差 | 加半熟半新的新信息 |
| `over_stimulation` | 玩家说无聊，其实是被复杂度劝退 | 降噪、拆步、加熟悉锚点 |
| `habituation` | 老玩家已完全预测流程 | 换用法、换语境、换组合深度 |
| `low_agency` | 事情很多，但玩家影响不了 | 修复行动后果和选择重量 |
| `low_meaning` | 奖励很多，但不值得在乎 | 把反馈接回目标、角色、世界或社交意义 |

输出字段必须包含：

```yaml
optimal_stimulation_fit:
  band: too_low | optimal | too_high | uneven | unknown
  boredom_type: under_stimulation | over_stimulation | habituation | low_agency | low_meaning | mixed | unknown
  design_direction: add_semi_novelty | add_familiar_anchor | reduce_overload | anti_habituation | repair_agency | add_meaning | preserve_curve | unknown
```

## 第零点五步：判断自由能区间

先判断玩家处在什么预测误差区间，再决定该降噪、提质还是调频。

| 区间 | 玩家反馈 | 设计含义 | 下一步 |
| --- | --- | --- | --- |
| `too_low` | 淡、空、重复、没有新东西 | 可处理预测误差不足 | 在低 CLP 前提下增加小惊讶、有效选择或反馈意义 |
| `optimal` | 上头、想再试、每局有变化 | 预测误差适中且可归因 | 保护曲线，谨慎微调 |
| `too_high` | 看不懂、太累、随机、不公平 | 预测误差过高或不可解释 | 先降 CLP，减少噪声和不可归因反馈 |
| `unknown` | 证据不足 | 不能判断 | 写明未知项，设计最小验证 |

自由能区间不是科学测量结果，只是设计判断。它服务于最佳刺激窗口：`too_low` 自由能不一定要加快节奏，`too_high` 自由能也不一定要降低难度，关键是预测误差能否被玩家吸收、解释、行动和复盘。输出时必须标注 `theory_status: design_hypothesis`。

## 第一步：先降噪，检查 CLP

玩家看不懂时，任何高频事件都会变成噪音。诊断体验低谷时，先问玩家是否能清晰接收信息，而不是先加奖励、加怪、加剧情或加弹窗。

| 检查项 | 典型症状 | 设计动作 |
| --- | --- | --- |
| UI 层级 | 玩家不知道主目标或可操作对象 | 合并入口，突出下一步，减少同时闪烁 |
| 规则理解 | 玩家不知道为什么赢/输/变强 | 用一次操作一条反馈替代长说明 |
| 特效噪音 | 玩家看不到关键危险或收益 | 区分战斗信息层级，降低非关键信号亮度 |
| 打断感 | 弹窗、加载、对白切断操作意图 | 改为战斗中提示、结算卡或可跳过短包 |
| 术语负担 | 新名词过多，玩家无法形成预测 | 用玩家目标语言替代系统内部语言 |

如果 CLP 没降下来，不要进入调频。更多事件只会让玩家更累。

## 第二步：再提质，检查 SF / EB / AR

当玩家能看懂后，再检查瞬时体验的纵向质量。这里不是看内容量，而是看一个瞬间是否足够可感、可归因、有身体感和氛围。

| 维度 | 诊断问题 | 设计动作 |
| --- | --- | --- |
| SF | 操作和结果是否被清楚反馈 | 同步视觉、音效、镜头、震动和 UI 状态 |
| EB | 玩家是否感觉角色听手 | 优化输入响应、起手/命中/收招、镜头和命中停顿 |
| AR | 留白是否有质感 | 用音乐、环境音、光影、美术和 UI 风格保持世界自洽 |

提质的关键是同频共振。一个反馈点最好只传递一个核心信息，例如重击、暴击、危险、升级、解锁、剧情转折。多个通道同时说不同的话，会提高 CLP。

## 第二点五步：检查马尔可夫毯耦合

当问题表现为“不跟手”“反馈不清”“我做了但世界没反应”时，要把它拆成感官状态、行动状态和耦合断点。

| 断点 | 典型表现 | 优先动作 |
| --- | --- | --- |
| `latency` | 输入到动作或反馈之间有迟滞 | 缩短响应链路，前移轻反馈 |
| `noise` | 关键反馈被特效、UI、音效盖住 | 降低非关键信号，统一反馈层级 |
| `unclear_mapping` | 玩家不知道哪个行动导致哪个结果 | 让行动后 1-3 秒内出现可归因反馈 |
| `weak_agency` | 玩家觉得选择不改变世界 | 让行动改变状态、风险、收益路径或下一段体验 |
| `overload` | 太多感官通道同时轰炸 | 降低并发反馈，只保留核心信息 |

## 第三步：后调频，检查 MD/min

只有在信息清晰、反馈有质感之后，才讨论增加或重排有意义选择。MD/min 不是让玩家一直点，而是让玩家持续形成预测、行动、反馈、修正的循环。

| 问题 | 典型表现 | 设计动作 |
| --- | --- | --- |
| 频度不足 | 玩家 30-60 秒没做过真正选择 | 前移短期选择或目标变化 |
| 质量不足 | 选项不改变局势，只是小数值 | 让选择影响后两波、下一节点或收益路径 |
| 频度过高 | 玩家无法理解选择差异 | 合并弱选择，保留少而重的节点 |
| 曲线平 | 全程一样快或一样慢 | 建立弱事件、强事件、留白的浓度曲线 |

## 浓度曲线

好的体验浓度通常不是一条直线，而是一条有设计意图的曲线。割草、动作、短视频式买量素材可能需要高水平密度；恐怖、探索、叙事、模拟经营可能需要较低水平密度，但通过 SF、EB、AR 维持纵向浓度。

诊断时要输出：

```text
当前段落意图：高压 / 建立规则 / 放松观察 / 情绪沉淀 / 决策规划 / 爽感释放
最佳刺激窗口：too_low / optimal / too_high / uneven / unknown
无聊类型：under_stimulation / over_stimulation / habituation / low_agency / low_meaning / mixed / unknown
当前问题项：CLP / SF / EB / AR / MD/min
建议顺序：先判窗口 / 再降噪 / 再提质 / 后调频
最小改动：一个可上线动作
验证方法：观察指标和玩家行为
```

## 快速问题定位

| 玩家反馈 | 更可能的问题 | 不要先做 |
| --- | --- | --- |
| 无聊但也看不懂 | over_stimulation 或 CLP 过高 | 加更多事件 |
| 老玩家疲劳 | habituation | 只加数值奖励 |
| 看不懂 | CLP 过高 | 加更多事件 |
| 没爽感 | SF 不足或不同频 | 只加数值奖励 |
| 不跟手 | EB 断裂 | 加剧情解释 |
| 世界很空 | AR 不足 | 只提高刷怪频率 |
| 没事做 | MD/min 低 | 加弱弹窗 |
| 太累 | CLP 高或 MD/min 过高 | 继续提速 |

## 输出要求

每次诊断都要先落到 `optimal_stimulation_fit`，再落到公式项。不要只写节奏慢、反馈弱、浓度低。要写清玩家是刺激不足、刺激过载、习惯化、低能动性、低意义感，还是证据不足；再写清是分母过高，还是分子不足，还是水平密度和设计意图不匹配。
