# 体验浓度公式旋钮玩法库

本文件把 ED 问题转成可上线的小改动。每次实验只选一个主旋钮，最多加一个不会干扰归因的辅助动作。

默认优先级是：先判断最佳刺激窗口，再 `CLP`，再 `SF/EB/AR`，最后 `MD/min`。

## OLSO 辅助定位

先判断玩家反馈属于哪一种刺激窗口，再选择 ED 主旋钮。

| 判断 | 设计含义 | 更可能的主旋钮 |
| --- | --- | --- |
| `under_stimulation` | 刺激太低，玩家没有可处理的新信息 | `MD/min`、`SF` 或 `AR` |
| `over_stimulation` | 玩家说无聊但其实是过载、太累或看不懂 | `CLP`，必要时加熟悉锚点 |
| `habituation` | 老玩家已经完全预测流程，模型不再更新 | `MD/min`、系统组合深度、语境变化 |
| `low_agency` | 事情很多但玩家影响不了世界 | `MD/min` 或 `EB` |
| `low_meaning` | 奖励/反馈不接目标、角色或世界意义 | `SF` 或 `AR` |
| `uneven` | 新手、回流、核心玩家的窗口不同 | 分群变体，不用全局平均节奏 |

这些判断只决定方向，不能省略公式项和埋点验证。

## FEP / 马尔可夫毯辅助定位

在 OLSO 之后，判断自由能区间和边界耦合，再选择 ED 主旋钮。

| 判断 | 设计含义 | 更可能的主旋钮 |
| --- | --- | --- |
| `too_low` | 可处理预测误差不足，玩家无聊 | `MD/min` 或 `SF` |
| `too_high` | 预测误差过高或不可解释，玩家崩溃 | `CLP` |
| `optimal` | 预测误差适中，玩家沉浸 | 保持曲线，小幅提质 |
| `latency` | 行动状态到反馈状态迟滞 | `EB` |
| `noise` | 感官状态噪声遮盖关键信号 | `CLP` 或 `SF` |
| `unclear_mapping` | 玩家不知道哪个行动造成哪个结果 | `SF` |
| `weak_agency` | 行动不能有效改变世界 | `MD/min` 或 `EB` |

这些判断只帮助选主旋钮，不能让一个变体同时改多个系统。

## CLP：降低认知负荷

适合处理看不懂、太累、UI 混乱、教学过长、特效光污染、目标不清、弹窗打断和术语负担。

| 改动 | 适合场景 | 影响窗口 | 风险 |
| --- | --- | --- | --- |
| 合并新手期同时闪烁入口 | 玩家不知道点哪里 | 前 1-3 分钟 | 功能曝光变慢 |
| 把长说明改成一次操作一条反馈 | 教程堆叠 | 首次使用 | 玩家漏信息 |
| 降低非关键特效亮度 | 看不见危险或收益 | 战斗中 | 爽感变弱 |
| 弹窗改为结算卡或可跳过短包 | 操作被打断 | 30-120 秒 | 信息到达率下降 |
| 内部术语改成玩家目标语言 | 规则不清 | 首次理解 | 精确性下降 |
| 目标链只突出下一步 | 多任务压迫 | 当前 checkpoint | 内容显得少 |

设计要点：CLP 是分母。玩家看不懂时，任何增加事件的方案都可能变成噪音。

## SF：提升可感知反馈

适合处理不爽、不清楚、反馈弱、打击无感、奖励无感、选择后无法归因。

| 改动 | 适合场景 | 影响窗口 | 风险 |
| --- | --- | --- | --- |
| 暴击同步特效、音效、镜头和数字层级 | 打击感弱 | 即时 | 光污染 |
| 升级或解锁增加短强反馈 | 成长无感 | 5-20 秒 | 奖励疲劳 |
| 选择后 10 秒内给结果预告 | 归因弱 | 10-30 秒 | UI 复杂 |
| Boss 掉落改成一次清晰仪式 | 关键奖励无记忆点 | 30-60 秒 | 节奏停顿 |
| 失败原因用一条可操作反馈表达 | 玩家不知道为何失败 | 下一次尝试 | 挫败感上升 |

设计要点：反馈不是越多越好，而是同频共振。多个感官通道要传递同一个核心信息。

## EB：提升具身感

适合处理不跟手、操作软、镜头晕、动作节拍不清、玩家觉得自己只是在按按钮。

| 改动 | 适合场景 | 影响窗口 | 风险 |
| --- | --- | --- | --- |
| 缩短输入到动作反馈的延迟 | 操控迟钝 | 即时 | 动作重量变轻 |
| 调整起手-命中-收招节拍 | 打击不稳 | 1-3 秒 | 动画返工 |
| 增加轻量命中停顿 | 命中无感 | 即时 | 节奏变卡 |
| 镜头保持目标和危险同屏 | 战斗混乱 | 战斗段 | 镜头表现变保守 |
| 手柄/移动端触觉只绑定关键反馈 | 震动噪音 | 即时 | 设备差异 |

设计要点：具身感不是剧情代入，而是意图被身体化执行的感觉。

## AR：提升氛围感

适合处理世界很空、留白无质感、风格割裂、音画不统一、UI 和世界气质不一致。

| 改动 | 适合场景 | 影响窗口 | 风险 |
| --- | --- | --- | --- |
| 给低频段加入环境音和动态氛围层 | 跑图或等待空 | 30-180 秒 | 性能和资源成本 |
| 统一 UI 动效、音效和美术语言 | 风格割裂 | 全流程 | 调整范围变大 |
| 用光影和粒子提示状态变化 | 空间反馈弱 | 当前场景 | 信息误读 |
| 结算卡承接世界观和情绪 | 打完后断裂 | 下一局 | 卡片噪音 |
| 留白段保留可观察的世界反应 | 没事做但需要沉浸 | 30-120 秒 | 节奏变慢 |

设计要点：氛围感让留白不空，但不能替代目标清晰和可操作反馈。

## MD/min：调整有意义选择频率

适合处理没事做、选择太弱、首个有效选择太晚、循环平铺、重复操作没有后果。

| 改动 | 适合场景 | 影响窗口 | 风险 |
| --- | --- | --- | --- |
| 首个有效选择前移到 30-90 秒 | 新手进入后空 | 前 3 分钟 | 教程过急 |
| 弱选项合并成少而重 | 菜单太多但无差异 | 下一段玩法 | 内容量看似变少 |
| 选项影响后两波敌人或下一节点 | Build 不成形 | 60-180 秒 | 难度波动 |
| 短期爽感 vs 长期收益 | 选择没有记忆点 | 1-3 分钟 | 认知负担 |
| 战斗后 5 秒给短链目标 | 打完不知道干什么 | 下一段行动 | 任务噪音 |
| 中段插入低成本稀有小事件 | 15-30 分钟疲劳 | 下一局/下一段 | 破坏经济 |

设计要点：MD/min 不是点击频率，也不是选项数量。选择必须改变局势、收益路径、表达、风险或接下来一段体验。

## Anti-habituation：反习惯化

适合处理老玩家疲劳、赛季中后段疲软、刷子/肉鸽重复、长期日常钝化。它不是单纯“多加内容”，而是让熟悉系统重新产生可学习差异。

| 杠杆 | 适合场景 | 小改动 | 风险 |
| --- | --- | --- | --- |
| alternative_use | 老系统只有一种最优用法 | 让旧技能/资源在新约束下有替代用途 | 破坏平衡 |
| attention_investment | 玩家无脑执行流程 | 给关键选择增加可观察线索和短期后果 | 提高 CLP |
| conscious_reframing | 玩家对旧内容形成固定标签 | 用任务、叙事或敌人目标重构旧系统意义 | 显得牵强 |
| context_shift | 同一动作在不同语境下无差异 | 换地图约束、敌人组合、资源稀缺性 | 过度随机 |
| combinatorial_depth | 系统之间没有新组合 | 增加少量可解释联动，而不是堆词条 | 核心玩家垄断理解 |

如果问题是 `habituation`，实验方案必须输出 `anti_habituation_plan`，并说明该杠杆如何保持半熟半新、可归因、可学习。

## 辅助动作

| 辅助动作 | 主要服务项 | 注意 |
| --- | --- | --- |
| narrative_compression | CLP / SF / AR | 压缩停顿和重复解释，不是删剧情 |
| objective_chain | CLP / MD/min | 只突出下一步，不要任务轰炸 |
| reward_clarity | SF / CLP | 让奖励价值被理解，不要纯数值膨胀 |
| recovery_window | MD/min / EB | 降低失败退出，不要抹掉挑战 |
| friction_trim | CLP / EB | 减少加载、确认、重复点击，不删除必要选择 |
| controllable_surprise | MD/min / SF | 增加可解释的小惊讶，不增加不可归因随机 |
| blanket_coupling_repair | SF / EB / CLP | 修复行动-反馈断点，优先降延迟、降噪声、提解释性 |

## 品类起手配方

| 品类 | 默认优先项 | 起手改法 |
| --- | --- | --- |
| 竖屏割草 ARPG | CLP -> SF -> MD/min | 先保证战斗信息可读，再强化清屏/暴击反馈，最后前移 Build 方向选择 |
| 回合制 Roguelike 卡牌 | CLP -> MD/min | 先让选项差异可理解，再让选牌/祝福影响后两场或下一节点 |
| 轻量塔防/防守 | MD/min -> SF | 白天准备和夜间压力之间增加短链反馈，避免等待空窗 |
| 叙事冒险 | CLP -> AR -> SF | 首段设定压进操作和环境，减少不可交互对白，同时保留情绪承诺 |
| 模拟经营 | AR -> MD/min | 留白有氛围，每 1-2 分钟给一个生产链状态变化或新目标 |
| 动作游戏 | EB -> SF -> MD/min | 先打磨操控和命中，再强化反馈，最后调整敌人/风险选择频率 |

## 禁止用法

不要用虚假倒计时、误导奖励、强制红点、付费焦虑、不可逆损失伪装或未说明的概率操控来提高指标。不要把数值奖励加倍包装成体验浓度优化。不要在一次实验里同时改奖励、难度、剧情、UI、关卡和商业化。
