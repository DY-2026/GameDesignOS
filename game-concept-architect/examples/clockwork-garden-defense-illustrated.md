# Illustrated Case: Clockwork Garden Defense

![Clockwork Garden Defense key art](../assets/clockwork-garden-defense-key-art.png)

## 一句话创意

> 做一个在废弃温室里培育机械植物、抵御夜间虫群的轻量策略防守游戏。

这不是一个“温室塔防换皮”。它要验证的设计核是：玩家在白天对机械植物进行有限调校，夜晚观察这些调校如何改变防线结果，并在下一天修正自己的培育策略。

## 4. Player Promise

### 对外宣传承诺

> 在一座废弃温室中培育会变形的机械植物，用有限白昼资源撑过一夜又一夜的虫群入侵。

### 前 10 分钟承诺

- 玩家会种下 3 种机械植物。
- 玩家会在攻击、减速、修复三种调校方向之间做取舍。
- 玩家会经历一次夜袭，看到防线在哪一格被突破。
- 玩家会在第二天主动改变调校顺序或布局。

### 长期游玩承诺

- 玩家会掌握不同植物协同。
- 玩家会形成自己的温室布局风格。
- 玩家会追求“用更少资源撑过更危险夜晚”的 mastery。
- 玩家会期待新植物、新虫群和新温室设施带来的策略变化。

## 5. Design Nucleus

> 白天的有限培育顺序会改变机械植物夜间功能，让玩家在“现在强化哪株植物”和“夜晚哪条防线最危险”之间做持续取舍。

为什么这是设计核：

| 判定项 | 说明 |
| --- | --- |
| 改变玩家行为 | 玩家不是只摆塔，而是要安排调校顺序 |
| 改变玩家选择 | 同一株植物可向攻击、减速、修复分支变化 |
| 改变节奏 | 白天规划，夜晚观察，第二天修正 |
| 可被原型验证 | 只需 3 种植物、2 类虫群、2 个夜晚即可测试 |

## 6. Core Loop

### Moment Loop

```text
查看格位状态 -> 选择植物 -> 投入白昼能量 -> 看到植物形态变化 -> 预判夜晚效果
```

### Session Loop

```text
查看虫群预告 -> 安排植物和调校顺序 -> 夜晚自动防守 -> 观察突破点 -> 获得零件 -> 修正下一天布局
```

### Meta Loop

```text
解锁植物模块 -> 形成温室流派 -> 挑战更长夜晚 -> 记录稳定防线方案
```

## 8. Key Systems

| 系统 | 服务循环 | 改变的玩家行为 | 反馈 | 验证方式 | 删除条件 |
| --- | --- | --- | --- | --- | --- |
| 白昼能量 | moment/session | 迫使玩家决定先调校哪株植物 | 能量消耗、植物亮度变化 | 观察玩家是否停顿比较 | 玩家无脑平均分配则重做 |
| 植物变形 | moment/session | 让同一植物承担不同职能 | 形态、攻击方式、音效变化 | 首局后询问玩家是否理解差异 | 只被理解为美术变化则重做 |
| 虫群预告 | session | 让玩家提前针对风险布局 | 入口提示、虫型图标 | 看布局是否响应预告 | 玩家无视预告则强化 |
| 夜间自动防守 | session | 让玩家观察自己白天决策的结果 | 路径、伤害数字、破防动画 | 记录失败归因 | 玩家觉得随机则重做因果展示 |
| 温室设施 | meta | 让玩家形成长期布局风格 | 喷灌、导轨、反光镜等设施效果 | Vertical Slice 阶段测试 | 如果抢走植物核心则后置 |

| 维度 | 当前判断 | 风险 | 需要验证 |

## 12. Scope Gate

| 层级 | 功能 / 内容 | 保留理由 | 验证目标 | 不做的代价 |
| --- | --- | --- | --- | --- |
| MVP 必须有 | 6x4 温室、3 种植物、2 类虫群、昼夜循环、白昼能量、夜晚反馈 | 没有它无法验证设计核 | 白天调校是否影响夜晚结果 | 画面表现会比较简陋 |
| Vertical Slice 应该有 | 植物变形动画、温室设施、夜间突发虫群、结算界面 | 展示完整体验承诺 | 玩家是否愿意复玩并优化布局 | 内容仍偏少 |
| Demo 后再做 | 多温室章节、稀有植物谱系、天气、剧情探索 | 依赖核心循环成立 | 扩展长期目标 | 暂时缺少内容丰富度 |
| 宣传概念可以保留但暂不开发 | 失落园艺协会、玻璃穹顶、植物图鉴故事 | 有氛围价值 | 不进入 MVP | 世界观表达较弱 |
| 建议砍掉的危险设计 | 开放世界采集、实时多人协作、复杂自动化工厂 | 成本高且偏离核心 | 避免 scope 失控 | 少一些大而全卖点 |

## 14. Validation Plan

### 最小可玩原型

- 原型形式：top-down graybox。
- 时长：10 分钟。
- 核心循环：白天调校植物，夜晚观察防守结果，第二天修正。
- 必须实现：温室网格、白昼能量、3 种植物、2 类虫群、2 个夜晚、失败归因提示。
- 明确不实现：剧情、商业化、多温室、完整美术、复杂音效、移动端适配。

### 第一轮测试目标

- 测试对象：5 到 6 名轻策略或塔防玩家。
- 样本数量：第一轮 5 人即可。
- 观察方式：录屏 + 操作观察 + 结束后 5 分钟访谈。
- 记录指标：第二局是否改变布局、是否能说清失败原因、是否愿意再玩一局。

### 最危险假设

| 假设 | 为什么危险 | 验证方式 |
| --- | --- | --- |
| 植物调校顺序能形成有意义策略 | 如果不成立，游戏只是普通塔防 | 观察玩家是否主动改变调校顺序 |
| 夜晚自动防守能提供清晰反馈 | 如果不清晰，玩家无法学习 | 让玩家复述哪条防线为何失守 |
| 3 种植物足够形成差异 | 如果不成立，MVP 无法判断核心 | 记录玩家是否使用不同布局 |

### 通过标准

- 5 名测试者中至少 4 名能说清“白天哪个决策导致夜晚结果”。
- 至少 3 名测试者第二局主动改变布局或调校顺序。
- 至少 3 名测试者愿意再玩一局，并能说出想尝试的新策略。

### 失败标准

- 多数玩家认为夜晚结果随机。
- 多数玩家只按提示操作，没有形成自己的布局判断。
- 玩家只能记住“机械植物题材”，说不出玩法差异。

### 下一步投入条件

只有当因果理解和第二局修正行为成立后，才进入 Vertical Slice。

下一步投入优先级：

1. 加强植物变形表现。
2. 增加 1 个温室设施。
3. 增加 1 类特殊虫群。
4. 做 1 个可展示的夜晚险胜高光。
