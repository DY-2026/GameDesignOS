# Feature Priority Matrix

| 功能 | 服务的承诺 | 服务循环 | 改变的玩家行为 | 反馈 | Unknown 降低 | 制作成本 | 依赖风险 | 优先级 | Scope 层级 |
| --- | --- | --- | --- | --- | --- | --- | --- | --- | --- |
|  | 对外/前10分钟/长期 | moment/session/meta |  |  | 高/中/低 | 高/中/低 | 高/中/低 | P0/P1/P2/P3 | MVP/VS/Demo后/宣传保留/砍掉 |

## 优先级定义

| 优先级 | 定义 |
| --- | --- |
| P0 | 不做就无法验证设计核 |
| P1 | 强化核心体验，适合进入 Vertical Slice |
| P2 | 有价值，但应等核心循环验证后再做 |
| P3 | 包装、扩展或宣传概念，暂不开发 |
| Cut | 成本或风险高，且不直接服务核心承诺 |

## 使用规则

- 每个功能都必须对应一个玩家承诺。
- 每个功能都必须说明改变哪类玩家行为。
- 不能解释验证方式的功能，不能进入 P0。
- 只服务世界观氛围的功能，默认进入 P3 或 Cut。
- 依赖大量内容生产的功能，默认不进入 MVP。
