# Assumption Ledger

凡是用户没有明确提供的信息，都必须写入本表。不要把 assumption 写成确定事实。

| ID | Assumption | 类型 | 来源 | 置信度 | 影响 | 验证方式 | 截止点 | 决策后果 |
| --- | --- | --- | --- | --- | --- | --- | --- | --- |
| A1 |  | 平台/商业/受众/制作/玩法/内容 | 用户未说明/合理推断/竞品经验 | 高/中/低 | 高/中/低 |  |  |  |

## 类型说明

| 类型 | 例子 |
| --- | --- |
| 平台 | 假设目标是微信小游戏，而不是 Steam |
| 商业 | 假设依赖广告或 IAP，而不是买断 |
| 受众 | 假设面向轻度玩家或策略玩家 |
| 制作 | 假设团队能做物理手感、联网或 3D |
| 玩法 | 假设玩家喜欢某种风险取舍 |
| 内容 | 假设有足够关卡、角色或剧情产能 |

## 置信度

| 等级 | 定义 |
| --- | --- |
| 高 | 用户明确提供，或有强约束支持 |
| 中 | 从创意和平台可合理推断 |
| 低 | 只是为了继续推演而临时设定 |

## 影响

| 等级 | 定义 |
| --- | --- |
| 高 | 错了会改变核心循环、平台、商业模式或 scope |
| 中 | 错了会影响系统优先级或验证方式 |
| 低 | 错了主要影响包装、命名或后续内容 |
