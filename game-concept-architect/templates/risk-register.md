## 风险类型

| 类型 | 说明 |
| --- | --- |
| 玩法 | 核心循环不好玩，选择不成立，反馈不清晰 |
| 平台 | 输入、局长、性能、加载、信息密度不适配 |
| 商业 | 广告、IAP、买断或运营方式破坏承诺 |
| 制作 | 团队能力、技术成本、美术成本、工具链不足 |
| 内容 | 需要过多关卡、角色、剧情、活动才能成立 |
| 合规 | 平台审核、未成年人、付费、隐私、版权风险 |
| 市场 | 差异化不足，受众不清晰，获量困难 |

## 风险动作

| 动作 | 使用场景 |
| --- | --- |
| 验证 | 风险高但可通过小原型判断 |
| 降 scope | 有价值但当前成本过高 |
| 替代方案 | 保留承诺，换低成本系统 |
| 砍掉 | 不服务核心循环或验证代价过高 |
| 暂缓 | 依赖后续数据或团队能力 |
