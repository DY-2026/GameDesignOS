# Core Loop Expansion

核心循环是玩家反复经历的行为闭环。它必须体现玩家承诺，并能解释为什么玩家会继续下一步。

## 循环层级

| 层级 | 时间尺度 | 作用 |
| --- | --- | --- |
| Moment loop | 3 到 30 秒 | 操作、即时选择、反馈 |
| Session loop | 3 到 15 分钟 | 一局目标、风险升级、奖励结算 |
| Meta loop | 多局 | 解锁、构筑、角色成长、策略表达 |

## 循环结构

```text
目标 -> 行动 -> 选择 -> 风险 -> 反馈 -> 奖励 -> 新约束 -> 下一轮目标
```

每个环节都要具体：

- 目标：玩家当前想达成什么？
- 行动：玩家按什么、拖什么、选什么、布置什么？
- 选择：玩家在什么选项之间取舍？
- 风险：错误、贪心、拖延或路线选择会带来什么损失？
- 反馈：玩家如何看见结果？
- 奖励：资源、能力、信息、空间、观众、评分还是剧情推进？
- 新约束：奖励是否改变下一轮决策，而不是只加数字？

## 系统扩展规则

新增系统必须填写：

| 字段 | 说明 |
| --- | --- |
| 服务的核心循环 | moment / session / meta 中哪一层 |
| 改变的玩家行为 | 玩家会因此怎么改变操作、选择、节奏、成长或表达 |
| 反馈 | 玩家如何知道这个系统有效或失败 |
| 验证方式 | 最小原型如何证明它有价值 |
| 删除条件 | 如果测试不成立，什么时候砍掉它 |

## 系统映射模板

```markdown
| 系统 | 服务循环 | 改变的玩家行为 | 反馈 | 验证方式 | 删除条件 |
| --- | --- | --- | --- | --- | --- |
|  |  |  |  |  |  |
```

## 从承诺到循环

示例承诺：

```text
玩家一边逃命一边制造直播效果，越危险越赚钱。
```

可能循环：

```text
躲避威胁 -> 选择是否做高风险表演 -> 热度上升或受伤 -> 获得打赏 -> 购买生存道具或节目效果 -> 威胁升级 -> 再次选择风险
```

此处设计核不是“直播题材”，而是“生存效率和节目效果互相冲突”。

## 常见错误

- 只有行动，没有选择。
- 只有奖励，没有新约束。
- 只有 meta 成长，没有 moment 乐趣。
- 把系统写成菜单列表，而不是行为变化。
- 所有系统都服务长期养成，前 10 分钟没有体验。
