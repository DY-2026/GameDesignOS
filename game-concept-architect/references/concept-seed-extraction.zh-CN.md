# Concept Seed Extraction

目的：把一句话创意拆成可设计、可验证、可删减的种子，而不是直接扩写成大段 GDD。

## 输入处理

先保留用户原句，再拆出三类信息：

| 类型 | 说明 | 例子 |
| --- | --- | --- |
| 明确信息 | 用户直接说出的内容 | “微信小游戏”“动物末世”“弹珠肉鸽” |
| 合理推断 | 从语义可推断但仍不确定 | “短局”“轻量操作”“可重复构筑” |
| 关键 unknown | 不知道就会影响设计方向 | 目标平台、商业模式、团队规模、参考游戏边界 |

## 必填字段

| 字段 | 要回答的问题 | 常见错误 |
| --- | --- | --- |
| 题材母体 | 这个创意借用了什么叙事、视觉、文化或身份材料？ | 把题材当玩法 |
| 玩法母体 | 它借用了什么操作、选择、循环或品类结构？ | 只写品类名，不写玩家动作 |
| 情绪承诺 | 玩家最应该感到什么？紧张、荒诞、掌控、贪婪、爽感？ | 写成世界观氛围，不落到行为 |
| 差异化种子 | 哪个冲突或机制会改变玩家行为？ | “题材新”但玩法不变 |
| 平台假设 | 适合 PC、移动、微信小游戏、Web、主机还是多端？ | 不考虑输入、时长、分发 |
| 商业化假设 | 买断、广告、IAP、订阅、赛季、UGC、B2B 还是品牌传播？ | 默认“以后再说商业化” |
| 受众假设 | 先吸引哪类玩家？他们已有的参照物是什么？ | 泛泛写“休闲玩家” |
| 关键 unknown | 目前最缺什么信息？ | 把未知写成确定事实 |

## 设计核判定

设计核必须至少命中一项：

- 改变玩家选择：玩家需要在两个有代价的行动之间取舍。
- 改变玩家节奏：玩家必须在时间、压力、资源或机会窗口中行动。
- 改变玩家成长：玩家的 build、能力、关系或地图状态会持续演化。
- 改变玩家表达：玩家能形成可识别的风格、路线、表演或策略身份。
- 改变反馈结构：玩家能明确感到某个行为被系统奖励、惩罚或扭曲。

不是设计核的内容：

- 题材标签。
- 世界观设定。
- 美术风格。
- “像某某游戏”。
- “加 Roguelike / 加养成 / 加社交”。

## 提取步骤

1. 把原句拆成名词、动词、情绪词、平台词、商业词、参照物。
2. 找出最强冲突：玩家想做 A，但系统逼他同时考虑 B。
3. 把冲突改写成玩家行为句：玩家每局会反复做什么取舍？
4. 标出哪些部分只是包装，哪些部分会真的改变行为。
5. 给每个未知项标注影响等级：高 / 中 / 低。

## 输出格式

```markdown
## Concept Seed Extraction

| 字段 | 结果 | 确定性 | 影响 |
| --- | --- | --- | --- |
| 题材母体 |  | 高/中/低 | 高/中/低 |
| 玩法母体 |  | 高/中/低 | 高/中/低 |
| 情绪承诺 |  | 高/中/低 | 高/中/低 |
| 差异化种子 |  | 高/中/低 | 高/中/低 |
| 平台假设 |  | assumption | 高/中/低 |
| 商业化假设 |  | assumption | 高/中/低 |
| 受众假设 |  | assumption | 高/中/低 |
| 关键 unknown |  | unknown | 高/中/低 |
```

## 质量检查

如果差异化种子无法写成玩家行为变化，则创意仍停留在包装层，需要重新提炼。
