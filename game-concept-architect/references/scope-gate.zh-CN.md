# Scope Gate

Scope gate 用于把想法拆成可以立刻做、稍后做、只保留宣传、应该砍掉的层级。

目标不是保守，而是保护设计核能被验证。

## 五层分类

| 层级 | 定义 | 判断问题 |
| --- | --- | --- |
| MVP 必须有 | 没有它就无法验证设计核 | 砍掉后，核心循环还存在吗？ |
| Vertical Slice 应该有 | 能展示完整体验承诺，但不是最小验证必需 | 它是否让体验更像最终产品？ |
| Demo 后再做 | 有价值，但当前会拖慢验证 | 它是否依赖核心循环已被证明？ |
| 宣传概念可以保留但暂不开发 | 对 pitch 有帮助，但原型不需要 | 它能帮助对外理解，但不影响测试吗？ |
| 建议砍掉的危险设计 | 成本高、验证弱、容易偏离设计核 | 它是否只是因为“听起来完整”才被加入？ |

## 评分方法

给每个功能打 1 到 5 分：

| 指标 | 说明 |
| --- | --- |
| 承诺贡献 | 是否直接兑现玩家承诺 |
| 行为改变 | 是否改变玩家选择、节奏、成长或表达 |
| Unknown 降低 | 是否能验证最危险假设 |
| 制作成本 | 实现、内容、调试、运营成本 |
| 依赖风险 | 是否依赖大量未完成系统 |

优先做：

```text
高承诺贡献 + 高行为改变 + 高 unknown 降低 + 低制作成本 + 低依赖风险
```

## 必砍候选

优先怀疑这些内容：

- 开放世界。
- 多人实时。
- 大量剧情分支。
- 复杂经济系统。
- UGC 平台。
- 全量角色养成。
- 大规模 LiveOps。
- 高精度物理手感。
- 高成本 3D 表现。

除非它们正是设计核，否则先进入 `Demo 后再做` 或 `建议砍掉`。

## 常见失败

- MVP 里塞进完整商业化。
- Vertical Slice 变成半个正式版。
- Demo 需要大量内容填充才能有趣。
- 宣传概念被误当成立刻要开发的功能。
- 没有写“建议砍掉”，说明还没有真正做取舍。
