# External Feasibility Scan

目的：在相信“绝妙点子”之前，先寻找外部证据，判断它是否已有需求、已有竞品、已有玩家动机，或只是创作者自我兴奋。

核心态度：

> 不要先幻想完整产品。先验证外部世界里有没有信号。

## 扫描对象

| 对象 | 要看什么 | 产出 |
| --- | --- | --- |
| 视频平台 | 高播放、评论关键词、收藏转发、反复观看理由 | 玩家动机关键词 |
| 商店平台 | 标签、竞品、评价、价格、销量估计、更新节奏 | 市场参照 |
| 社区讨论 | 玩家抱怨、未满足需求、mod、二创、攻略 | 需求缺口 |
| 搜索趋势 | 搜索指数、节日波峰、地域差异 | 需求时机 |
| 参考作品 | 相近体验、可借鉴结构、不可复制部分 | 对标边界 |

## 评论和反馈提取

不要只看“好玩”。要提取玩家为什么被吸引。

| 维度 | 示例问题 |
| --- | --- |
| 情境 | 玩家在什么时间、节日、生活场景中最想体验它？ |
| 记忆 | 它是否连接童年、青春、关系、仪式、怀旧或身份？ |
| 行动 | 玩家真正想亲手做什么？准备、建造、表演、管理、冒险？ |
| 缺口 | 玩家反复抱怨什么没有被满足？ |
| 二创 | 玩家是否愿意做 mod、视频、攻略、分享或挑战？ |
| 复玩 | 玩家为什么每年、每季、每次更新还回来？ |

## Evidence 状态

| 状态 | 定义 |
| --- | --- |
| `verified` | 已有当前来源、链接、评论或数据支持 |
| `partial` | 有间接证据，但还不足以支持立项 |
| `evidence-needed` | 当前只是推测，必须后续外部验证 |
| `contradicted` | 外部证据与创意假设冲突 |

## 使用规则

- 如果没有联网或没有来源，不要编造数据。
- 如果用户只要设计案，也要写明“外部验证状态”。
- 如果外部已有大量相似产品，不代表不能做，但必须说明差异化行为。
- 如果外部完全没有信号，不代表不能做，但必须提高风险等级。
- 如果玩家需求集中在情境、仪式、DIY、社区、怀旧、挑战等关键词，要把这些词转化为玩家承诺和核心循环。

## 常见失败

- 只看竞品数量，不看玩家为什么玩。
- 只看高播放，不看评论动机。
- 只看好评率，不看差评暴露的缺口。
- 只看题材热度，不看玩法是否能承接题材。
- 用旧数据或未查数据得出确定市场判断。
