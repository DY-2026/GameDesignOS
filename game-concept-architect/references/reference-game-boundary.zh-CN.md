# Reference Game Boundary

目的：当用户说“像某个游戏”时，只抽取可迁移的行为结构，不复制原作结构、术语、设定、阵营、角色、内容表达或商业策略。

## 边界字段

| 字段 | 说明 |
| --- | --- |
| `reference_game` | 用户提到或 agent 识别的参考作品 |
| `role_in_request` | 灵感来源、硬约束、市场参照、操作参照、情绪参照 |
| `extract_behavior_structure` | 可借鉴的玩家行为结构 |
| `do_not_copy` | 不复制的内容表达、设定、术语、阵营、角色或专有结构 |
| `difference_goal` | 本项目必须形成的差异化行为 |
| `risk` | 抄袭、错配、scope 膨胀或受众误判风险 |
| `validation` | 如何验证参考结构是否真的适合本项目 |

## 处理步骤

1. 先确认参考游戏在请求中的角色：灵感、约束、受众参照还是平台参照。
2. 把参考内容拆成行为结构，例如探索、撤离、构筑、协作、风险管理、表达。
3. 明确不保留的内容：世界观、专有术语、角色、阵营、剧情、关卡结构、UI 表达。
4. 写出本项目的差异化行为，而不是只换皮。
5. 把可迁移结构放进 design nucleus options 或 core loop，不要直接复制功能清单。

## 最小示例

```markdown
## Reference Game Boundary

| Reference | Role | Extract Behavior Structure | Do Not Copy | Difference Goal | Validation |
| --- | --- | --- | --- | --- | --- |
| 某撤离玩法作品 | 操作/风险参照 | 进入危险区、带资源撤离、失败损失推动紧张感 | 原作阵营、术语、地图结构、装备体系 | 把撤离压力转化为太空站结构变化下的路线决策 | 10 分钟原型测试玩家是否主动权衡继续探索或撤离 |
```

## 质量门

- 每个参考游戏都必须写 `do_not_copy`。
- 输出不得复制原作术语、阵营、设定、关卡结构或内容表达。
- 参考只能服务玩家行为结构，不能替代原创设计核。
- 如果参考游戏带来 scope 膨胀，必须进入 scope gate。
