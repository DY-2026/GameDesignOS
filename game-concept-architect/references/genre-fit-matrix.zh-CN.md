# Genre Fit Matrix

用途：当一句话创意包含品类、竞品或混合类型时，判断“玩法母体”是否真的适合玩家承诺和平台约束。

## 使用方法

1. 找出创意中的参照品类或参考游戏。
2. 不要复制表层系统，先提取玩家行为。
3. 检查该品类的默认节奏是否服务情绪承诺。
4. 检查平台输入、局长、商业模式是否冲突。
5. 标出最危险假设。

## 品类适配矩阵

| 品类母体 | 核心玩家行为 | 适合的承诺 | 主要风险 | 常用验证指标 | 平台注意 |
| --- | --- | --- | --- | --- | --- |
| Survivor-like | 移动规避、自动攻击、升级选择、压力堆叠 | 高压成长、爽感清屏、短局构筑 | 容易同质化，选择深度不足 | 3 分钟留存、升级选择犹豫率、死亡后再开率 | 移动端要控制操作负担 |
| Roguelite | 随机路线、build 选择、失败后再来 | 每局不同、成长 mastery | 随机性掩盖设计弱点 | 再开局率、build 识别度、失败归因清晰度 | 局长要匹配平台 |
| 弹珠 / Pachinko | 发射、碰撞、概率收益、路径预测 | 运气与策略混合、爽感结算 | 玩家可能感到不可控 | 发射前决策时间、结算理解度、复投欲望 | 适合触屏和短视频传播 |
| 弹球 / Pinball | 反应操作、角度控制、连击 | 手感 mastery、局内逆转 | 物理手感成本高 | 失误归因、连击达成率、手感评分 | 需要高帧率和清晰反馈 |
| Deckbuilder | 抽牌、选牌、删牌、构筑协同 | 策略组合、知识成长 | 学习门槛高、UI 信息重 | 卡牌理解时间、combo 发现率 | 小屏要简化信息密度 |
| Lane / Train defense | 分层布防、敌人路线、资源调度 | 空间策略、压力推进 | 容易变成静态数值比拼 | 换位频率、失败归因、关卡重玩率 | 适合短局，但 UI 需清晰 |
| Idle / Incremental | 投入资源、等待增长、重置放大 | 轻松成长、指数快感 | 核心行为弱，验证误导 | 回访率、升级点击价值感 | 商业化容易侵蚀公平感 |
| Simulation / Management | 规划、分配、排程、反馈优化 | 经营掌控、系统涌现 | 前期慢热，反馈滞后 | 首次目标完成率、瓶颈理解度 | 移动端要拆短任务 |
| Social / Livestream sim | 表演、观众反馈、热度经营 | 被观看、冒险、身份表达 | 观众系统可能只是数值皮肤 | 观众事件记忆度、表演选择率 | 适合短视频化传播 |

## 参考游戏使用边界

当用户说“像某某游戏”时，必须拆成：

- 借鉴的体验：玩家为什么喜欢它？
- 借鉴的结构：循环、节奏、选择、反馈是什么？
- 不借鉴的部分：哪些系统不适合当前平台和团队？
- 差异化行为：本项目让玩家做出什么不同选择？

## 输出模板

```markdown
## Genre Fit

| 参照母体 | 保留的行为结构 | 不保留的内容 | 与本创意的冲突 | 调整方案 |
| --- | --- | --- | --- | --- |
|  |  |  |  |  |
```

## 红线

- 不要把“像某某游戏”当作设计完成。
- 不要把题材换皮当作差异化。
- 不要在未验证核心循环前承诺复杂 meta。
- 不要忽略小屏、短局、触控、加载、断线、分享链路等平台现实。
