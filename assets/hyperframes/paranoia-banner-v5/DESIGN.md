# GameDesignOS Banner Visual Direction

## 生图方向

目标不是“技能清单海报”，而是一张能让 GitHub 读者一眼感到这是 `GameDesignOS` 的操作系统级封面：游戏创意、体验证据、验证实验、策划案和 agent workflow 被组织成同一个可操作的设计控制台。

## 可直接使用的英文 prompt

Premium dark technical command-center banner for GameDesignOS, an AI-native operating system for game design, concept validation, experience diagnosis, proposal writing, and prototype iteration. The scene should feel like a serious design control plane, not a prompt collection or skill-list poster: layered evidence streams, contract handoff lines, playable-loop diagrams, experiment gates, and proposal-review panels converge into one coherent operating surface. No critical text inside the generated background; leave clean space for a locally overlaid title. Cinematic but restrained, GitHub README scale, crisp interface geometry, balanced cyan, amber, and violet accents on deep ink background.

## 构图要求

- 宽幅 GitHub README banner，优先 16:9 或 1920x1080。
- 中央留出清晰标题区，供本地叠加 `GameDesignOS`。
- 背景表达五个系统层：concept、evidence、experiment、proposal、workflow governance。
- 画面应该像“游戏设计操作系统的控制面板”，不是赛博装饰图。
- 关键流程必须在 README Markdown 里另有文字说明，图片只承载氛围、结构隐喻和识别度。

## 本地文字层

- Title: `GameDesignOS`
- Signature: `by Paranoia`
- Subtitle: `Evidence, contracts, experiments, and proposals for AI-native game design.`
- Version badge uses a small cyan glass chip with soft glow.
- Title uses layered glow, subtle chromatic ghosts, gradient text fill, and a diagonal light sweep.

## 不要做

- 不要在生成图里放密集说明文字。
- 不要放会变成乱码的假 UI 小字。
- 不要放 mascot、人物、过度装饰或品牌侵权元素。
- 不要让亮色面板压住标题可读性。
- 不要做成大面积紫蓝渐变背景；cyan、amber、violet 要平衡。
