# Negative Cases

## 1. 通用 GDD 幻想文档

失败表现：用户要求商业游戏策划案，输出大量世界观、职业、地图、角色、技能和剧情，但没有文档受众、商业模式、scope gate、生产风险、验证标准和决策请求。

修正：先建立 proposal intake，再用玩家承诺、核心循环和投决问题重排章节。

## 2. 把独游写成手游商业案

失败表现：用户要求 Steam 独游设计案，输出签到、广告、抽卡、赛季、长期 LiveOps 和付费成长。

修正：切换到 `indie_design_dossier`，强调创作命题、商店页承诺、内容最小化、制作边界和 vertical slice proof。

## 3. 没有证据的市场确定判断

失败表现：写出市场已有验证、愿望单潜力很高、发行一定会感兴趣、买量成本可控，但没有当前来源和调研。

修正：改成 `needs_research` 或 `assumption`，说明 VOI 和最小调研动作。

## 4. Scope 失控

失败表现：First Playable 要同时做完整养成、剧情、多角色、多地图、社交、商业化、数据后台和新手引导。

修正：按 MVP、First Playable、Vertical Slice、Demo、Release、Post-launch、Parked、Cut 分层。

## 5. 只有建议，没有决策

失败表现：文档说了很多可以怎么做，但最后没有批准什么、投入什么、验证什么、何时停止。

修正：补 `Decision Request` 和 `Go/No-Go Criteria`。
