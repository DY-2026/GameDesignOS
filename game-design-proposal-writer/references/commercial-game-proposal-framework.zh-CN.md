# Commercial Game Proposal Framework

商业游戏策划案的目标是推动资源决策。它要回答：为什么值得做，做成什么算成立，下一步用多少资源验证什么。

## 文档主线

商业案不应该从系统列表开始，而应该从下面这条链路展开：

```text
目标玩家行为 -> 玩家承诺 -> 核心循环 -> 商业/平台适配 -> scope gate -> 验证指标 -> 投入决策
```

## 推荐章节

| 章节 | 要回答的问题 | 常见失败 |
| --- | --- | --- |
| Executive Summary | 这个项目一句话是什么，为什么现在值得看 | 只写题材，不写玩家行为 |
| Product Positioning | 属于什么品类/相邻品类，差异化在哪里 | 把题材混搭当差异化 |
| Target Player and Desire | 玩家想反复做什么，为什么兴奋，为什么反感 | 只写年龄、性别、泛标签 |
| Player Promise | 对外、前 10 分钟、长期分别承诺什么 | 先堆功能再补体验口号 |
| Core Loop and Key Systems | 玩家怎么行动、选择、反馈、奖励和成长 | 系统名很多但不回连循环 |
| Platform and Business Fit | 平台、渠道、商业模式如何影响体验 | 商业化和核心体验互相打架 |
| Scope Gate | MVP/VS/Demo/Release/Post-launch 怎么切 | MVP 接近完整正式版 |
| Production Feasibility | 团队、技术、内容产能、工具链是否匹配 | 未知团队被默认能做大项目 |
| Metrics and Validation | 用什么指标验证哪个假设 | DAU/留存/收入孤立出现 |
| Risk Register | 最大风险是什么，怎么降风险 | 风险写成泛泛而谈 |
| Decision Request | 现在需要评审者批准什么 | 只有方案，没有请求 |

## 商业适配检查

商业模式不是后加模块。它会改变设计边界。

| 模式 | 设计影响 | 风险 |
| --- | --- | --- |
| 买断 | 首屏卖点、完整体验、口碑、内容密度 | 内容成本过高，复玩不足 |
| F2P + IAP | 长期目标、成长节奏、价值分层 | 付费压力破坏玩家承诺 |
| 广告变现 | 短会话、失败/奖励节点、节奏切分 | 广告打断核心反馈 |
| 赛季/通行证 | 长期目标、内容更新节奏、任务结构 | LiveOps 产能不足 |
| UGC/社区 | 创作工具、分享链路、审核和生态 | 工具成本和冷启动过高 |
| 小游戏 | 低门槛、短会话、轻社交/传播 | 深循环和重操作不适配 |

没有当前证据时，只能写成 `business_model_assumption`。

## 指标必须回连假设

| 假设 | 可观察指标 |
| --- | --- |
| 玩家理解核心因果 | 首次关键行为完成率、第二局主动使用率、失败归因反馈 |
| 核心循环有复玩欲 | 第二局开始率、重复会话间隔、主动目标设置 |
| 商业化不破坏体验 | 广告后流失、付费点前后行为变化、负反馈关键词 |
| 长期目标可承载 | 中期目标完成率、内容消耗速度、疲劳点 |
| 社交/传播成立 | 分享触发率、回流率、被邀请者完成率 |

## 立项建议格式

每份商业案最后都应有明确请求：

```text
建议批准/不批准/延后/转向：
下一步投入：
验证周期：
验证对象：
通过标准：
失败标准：
若通过，进入：
若失败，执行：
```
